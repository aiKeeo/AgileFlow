# todo.md 模板

```markdown
# 项目待办事项

## 项目信息
- 项目名称：
- 创建时间：
- 预计完成时间：

## 流程进度
- [ ] 项目盘点（init，仅 brownfield）
- [ ] 需求澄清
- [ ] 数据建模
- [ ] 方案设计（sol:）
- [ ] 开发实现
- [ ] 测试验收

> **快速 / 严谨 / AI 自主均须维护本区勾选。**  
> 快速 ≠ 跳过上列阶段；AI 自主 ≠ 不写进度。建模按需跳过时标 `⏭️` 并写依据。

## 决策委派（可选 · 须与 agileflow.env 同步）

> 见 [stage-delegation.md](../templates/stage-delegation.md)  
> **机器权威**：`atlas/agileflow.env` 的 `AF_DECIDE` / `AF_FLOW` / `AF_PHASE`（闸门读这个；本区给人看，AI 改决策时**两处一起改**）。  
> **首启须流程启动卡**；禁止静默写成 AI自主/快速。用户选定后：`ai`=过程少问；`user`=逐步问。  
> **「用户不用管 / 后面都你定」写这里（AI自主），不要当成可以薄化下方开发任务。**

- 全局：未设置 | AI自主 | 用户决策
- 模式：未设置 | 快速 | 严谨
- 自阶段：— （例：`req` 起 AI自主）
- 备注：用户说「这阶段我来」→ 仅当阶段改回用户决策；同时 `AF_DECIDE=user`

## 模块配置（可选 · 覆盖默认档位）

> 见 [04-development §风险分档](../phases/04-development.md#风险分档)。不填则用默认。

- dev 文档档位：标准（默认）| 精简 | 完整
- 闸门档位：2道（默认）| 1道 | 3道+grep
- 停点控制：合并卡（默认）| AI自主 | 逐步停
- 变更管理：标准（默认）| lite | 完整5步

## 开发任务

> **每项必须拆成三步子项**；**① 未勾 → 禁止 ② 写码**。  
> **`atlas/todo.md` 三段式强制**（快速/严谨/AI自主均同）：每个任务 = `### T-xxx` + ①②③；**不是推荐、不可省略**。  
> **快速模式同样详细**：禁止因「快速」合并多 T、省略 ①②③、或写成「实现 MVP」一条。  
> **阶段 4 入口铁律**：必须用 **`TodoWrite` 为每一个 T 各建三条**（①构思 / ②写码 / ③AC）；**禁止**一条 Todo 覆盖多个 T；**禁止**只建①。  
> 父任务 `- [x]` = 三步全 ✅；全部父任务 ✅ → **AskQuestion 阶段闸门**。  
> 任务头格式：`### T-xxx：[端] 描述（估时）— 功能ID [/ 契约ID] [精简|标准|完整]`（**禁止**写成 `- [ ] T-xxx 描述` 一行；档位由风险维度定，见 [04-development §风险分档](../phases/04-development.md#风险分档)）  
> 端标记：`[BE]` 后端 | `[FE]` 前端 | `[FULL]` 不可拆分

### ① 质量门槛（勾①前 · **冻结区** · 阶段 3 写入后全程有效）

> **有文件 ≠ 过构思闸门。** 权威检查表 + 字面量速查 → **[dev-quickstart §构思闸门](dev-quickstart.md#构思闸门勾-①-前)**
> **冻结**：构思闸门标准由 skill 规定；**禁止**改成项目自定义规则（XSS/JDK/业务规则等写 `humanTodo` 或 REQ）。改写门槛 = **无效**，勾①仍须按原表 + 字面量校验。

### T-001：[BE] {后端任务简述}（{估时}）— F-001
- [ ] **① 构思落盘** → `atlas/dev/T-001-{简述}-BE.md`（Read exemplar → 按模板落盘 → **字面量校验过** → 才勾）
- [ ] **② 按 ## 做法 写码**
- [ ] **③ 对照 REQ 验收 AC**

### T-002：[FE] {前端任务简述}（{估时}）— F-001 / API-001
- [ ] **① 构思落盘** → `atlas/dev/T-002-{简述}-FE.md`（Read exemplar → 按模板+布局图 → **字面量校验过** → 才勾）
- [ ] **② 按 ## 做法 写码**
- [ ] **③ 对照 REQ 验收 AC**

## 功能依赖（阶段 3 填写，供阶段 4 串行排序）

| 功能 ID | 依赖 |
|---------|------|
| F-001 | — |
| F-002 | F-001 |
| F-003 | F-002 |

## 任务批次（可选，仅并行模式参考）

> **默认不用**。阶段 4 按上方「开发任务」列表**从上到下逐项**执行。  
> 仅用户显式「并行开发 / 同时开发 FE+BE / 多 subagent」时，主 Agent 按批次分组并读 [parallel-orchestration.md](../phases/parallel-orchestration.md)。

| 批次 | 功能 | 任务 | 依赖 | 共享风险 |
|------|------|------|------|----------|
| 0 | 基建 | T-000 | — | schema.sql, 路由 |
| 1 | F-001 | T-001~T-003 | — | 无 |

## 进行中
<!-- 跨会话 checkpoint：新聊天必读；每推进一步就更新 -->
- **checkpoint**：`T-xxx` · 步骤 `①|②|③` · 更新于 `YYYY-MM-DD`
- 无

## 已完成任务
- 无

## 变更历史
| 时间 | 操作 | 说明 |
|------|------|------|
```

## 状态标记

| 标记 | 含义 |
|------|------|
| 三步子项 `- [ ]` | 该步未开始/进行中 |
| 三步子项 `- [x]` | 该步已完成 |
| 父任务 `- [x]` | ①②③ 三步全 ✅ |
| 「进行中」checkpoint | **跨会话权威**：当前 **T-xxx** + 步骤 ①/②/③ |

---

## TodoWrite 强制展开（防漏① · 最高优先级）

> **记不住规范 → 靠清单。** 阶段 4 / 「全部开发」时，Agent **必须先**把每个 T 的构思步骤写进 **TodoWrite**，再一个个做。  
> **跨会话权威 = `atlas/todo.md`（含 checkpoint）**；TodoWrite 是**本会话工作台**，须从 atlas 重建。  
> **禁止**新聊天以空 TodoWrite / 记忆为准推进。

### 跨会话续作（resume · 强制）

新开聊天 / 「继续 agileflow」且已有开发任务时：

1. **Read** `atlas/todo.md`：流程进度、各 T 的 ①②③ 勾选、**「## 进行中」checkpoint**  
2. 用 TodoWrite **按 atlas 勾选状态重建**（已 `[x]` → completed；当前步 pending）  
3. 从 checkpoint 的 `T-xxx · 步骤` 继续；无 checkpoint → 取第一个未完成子项  
4. **每勾完一步**：同步改 `atlas/todo.md` **且**更新 checkpoint 行

**checkpoint 为步骤 ② 时**（②部分完成续作）：
1. 检查已有源码是否能编译
2. 可编译 → 继续完成②剩余部分 → 可运行闸门 → ③
3. 不可编译 → 评估已有代码可用性，必要时回退到①重写做法

**多会话并发保护**：更新 todo.md 前先 Read 最新版本，对比 checkpoint 时间戳。检测到 checkpoint 比当前会话最后读取的新 → AskQuestion「检测到另一会话可能在进行。确认继续？(是→合并 / 否→暂停)」。建议同一项目同一时间只开一个 Agileflow 会话。

### todo.md 损坏/缺失时的恢复

`atlas/todo.md` 被误删、格式损坏或 git 冲突时：

1. 扫描 `atlas/dev/T-*.md` 文件列表 → 推断任务清单（文件名含 T-xxx）
2. 扫描每个 dev 文件的 `## 结果` 段是否有可运行证据 → 推断 ③ 是否完成
3. 扫描 `atlas/requirements/` 和 `atlas/solution/` 的 README.md 状态 → 推断流程进度
4. AskQuestion：「检测到 todo.md 缺失/损坏。根据 atlas/ 文件推断当前状态：{推断摘要}。请确认或纠正。」
5. 用户确认后重建 todo.md → 继续 TodoWrite 重建

### 何时必须展开

命中任一 → **本回复内先按 atlas 重建/展开 TodoWrite，再写任何业务源码**：

1. 进入阶段 4  
2. 用户说「全部开发 / 直接全开发 / yes_all / 连续做」  
3. 从暂停恢复 / **新会话续作**  

### 必须建成的条目（每个 T 三条①②③）

| 要求 | 说明 |
|------|------|
| **1 T = 3 条 TodoWrite（强制）** | 每个 T 头 **必须**有 TodoWrite：`① 构思→atlas/dev/T-xxx…` + `② 写码 T-xxx` + `③ AC T-xxx`；**禁止**只建①不建②③ |
| **① 条目 content 必含** | 字面量 `①` + `构思` + 路径 `atlas/dev/T-xxx` + **`exemplar` 或 `构思闸门` 或 `字面量校验` 或 `grep`**（提醒对照范例+机械检查） |
| **id 命名** | `T-002-1` / `T-002-2` / `T-002-3`（或 `t002-dev`）；**禁止** `be-all` / `t002-018` |
| **顺序** | 未完成的 `T-xxx-①` 全部做完之前，**禁止**把任意 `T-yyy-②` 标 completed（显式并行批量构思 批量勾①除外，见 parallel；单批最多 3 T） |
| **勾 completed 条件（①）** | ① 文件存在（非 README/temp）；② **构思闸门 全过（含反假标题+字面量校验）**；③ **已对本文件 Grep** 且通用项（FE+UI 含布局/映射）命中；④ 回复含 `字面量校验：…已命中`。缺任一 → **禁止** completed |
| **勾 completed 条件（②）** | 该 T 的①已 completed；Write ⊆ `## 做法` |
| **勾 completed 条件（③）** | 该 T 的②已 completed；**可运行闸门 已过**（编译+能启/接口+冒烟，证据已写）；`## AC` 中可自动化 AC 已绿 |

### ❌ 禁止（洗清单）

```
❌ TodoWrite 一条：「实现 T-002~T-018」/「全部 API」/「后端+前端」
❌ 只建 T-001 三条，其余 T 不建①就写码
❌ 把多个 T 的①合并成「写完所有 dev」一条
❌ ① 未建文件就标 completed
❌ ① 有文件但未跑字面量校验 / 过短文档仍 completed
❌ 把 todo「①质量门槛」改成项目自定义规则当过关依据
❌ 「全部开发」时跳过 TodoWrite 直接开写
❌ atlas/todo 写成扁平列表：`- [x] T-001 脚手架`（无 ### T 头、无 ①②③）
❌ 只用 `atlas/dev/README.md` 索引代替每个 `atlas/dev/T-xxx-*.md`
```

### ✅ 正确（18 个 T 时）

```
TodoWrite（可分批 merge，但①必须先全部出现为 pending）:
  T-001-1 | ① 构思 → atlas/dev/T-001-…-BE.md（对照 exemplar / 构思闸门）  pending
  T-001-2 | ② 写码 T-001                          pending
  T-001-3 | ③ AC T-001                            pending
  T-002-1 | ① 构思 → atlas/dev/T-002-…-BE.md（对照 exemplar / 构思闸门）  pending
  …（每个 T 都有 ①②③ 三条）
```

「全部开发」= **按 TodoWrite 从上到下**做完每一条；**不是**跳过清单出代码。

## 规则

- 开发任务只维护此处；**一任务 = 一个 `### T-xxx`（或 `####`）头 + ①②③ 三步子项**（**强制**，非推荐）
- 阶段 3：拆解任务 + 功能依赖；**每个 T-xxx 须预填三步子项与 dev 路径**；不过 [方案自检四项](solution-architecture.md#阶段-3-结束前自检不通过禁止标方案已确认--禁止进阶段-4) 禁止确认方案
- 阶段 4：**先按 atlas checkpoint 重建 TodoWrite** → 再严格按子项顺序 ①→②→可运行闸门→③（细则 [04-development.md](../phases/04-development.md)）
- 阶段 4 开始 / 续作 **T-xxx** 时：**TodoWrite 从 atlas 同步**；勾选时**两边一起改**，并刷新 checkpoint
- **会话内**可看 TodoWrite 推进；**跨会话**只信 `atlas/todo.md`
- **禁止**：跳过 ① 直接写码；三步未全 ✅ 就把父任务标完成
- **禁止扁平列表**：`- [ ] T-001 描述` 一行完事（无三段式）= 阶段 3/4 违规
- **禁止合并**：不得写成 `### T-002 ~ T-013 ✅` 或 TodoWrite 一条「实现 T-002~013」；**一次只推进 1 个 T-xxx**（显式并行：可先齐多 T 的①，②仍 1 Task=1 T）
- **勾 ① 前**：对应 `atlas/dev/T-xxx-*.md` 必须已存在且过构思闸门（禁止「先勾再补文件」；**`dev/README.md` 不算①**）
- **标「开发实现 ✅」前自检**：
  1. 合法 T 头数（`^#{3,4} T-\d+`，**仅 ###/####**；`## T-` **不算**）≥1；若开发任务区出现 `T-\d+` 但合法 T 头=0 → **格式违规，禁止 ✅**
  2. **每个**合法 T 头下须有含字面量 `①` `②` `③` 的三行子项（① 行含 `atlas/dev/T-xxx`）；缺任一行 → **禁止 ✅**
  3. `atlas/dev/T-*.md` 个数（排除文首 `顺序：⚠️` 事后补写；**排除 README / temp/**）**=** 合法 T 头数
  4. TodoWrite 中每个 T 的①②③均为 completed；每 T 可运行闸门 有证据
- **事后补写**：先码后写的 dev 文首标 `顺序：⚠️ 先码后文档`；**禁止勾 ①**；父任务只可标 `⚠️ 事后补写归档`  
  - **禁止**把「开发实现」伪装成 ✅——**即使用户说「接受技术债 / 先这样」也不行**  
  - 要正式 ✅：必须**另写合规①**（无 ⚠️）→ 过构思闸门 → 再按 ②→可运行闸门→③ 重验（可复用已有源码，但流程不可跳）
- **勾 ③ 前**：须过 [可运行闸门](dev-quickstart.md#可运行闸门每-t--每模块强制)；禁止「单测绿但起不来」仍 ✅
- **「给用户看 / MVP」**：存在端编译+启动+主路径冒烟全过，才允许说可看
- **「全部开发」**：主 Agent 串行；先展开 TodoWrite；每 T 含可运行闸门；禁止 Subagent 一次写完多 T
