# 构思落盘文档（atlas/dev/）

> **格式模板**：颗粒度以 [dev-exemplar-BE.md](../examples/dev-exemplar-BE.md) / [dev-exemplar-FE.md](../examples/dev-exemplar-FE.md) 为准；落盘时换成本任务内容。
> 阶段 4 **步骤 ①**。  
> **dev = 构思落盘，不是事后补文档。**  
> **没落盘 → 禁止写业务源码。**

---

## dev 规则：构思先行

**开发的第一件事是构思，第二件事才是写码。** ① 须按下方**七段模板**完整输出到 `atlas/dev/`。

| 规则 | 说明 |
|------|------|
| **D1** | dev 文档 = **构思落盘**；聊天里的思路不算，必须写文件 |
| **D2** | 一任务一构思：`T-{id}-{简述}-{BE\|FE}.md` |
| **D3** | 严谨一～九标题；快速一/五/八/九（FE+三）；二四六七快速可整段省略 |
| **D4** | 五须可执行（做法含 `` `Class.method` `` 或 pages/services）；禁空壳/「见 REQ」独撑 |
| **D8** | FE+UI：字面量 `### 3.1` `### 3.2` `### 3.3`；3.2 英文字段 |
| **D5** | A0 Read exemplar→写1个T→A→①→B→②→③；**禁止批量①** |
| **D6** | 补盘/摘要/空壳 = 无效；不勾① |
| **D7** | ①未勾禁止写码 |
| **D9** | 禁合并多T；连续≠压缩 |
| **D10** | 闸门A：段标题+五可执行；T≥3或BE+FE强制严谨 |

architecture / contracts 讲**是什么**；**dev 讲这任务怎么构思、怎么写、怎么验**——不可互相替代。

细则：[04-development.md](../phases/04-development.md)

## 七段构思模板（① 必须按此格式输出）

> **照抄本节结构**；**五** 结构见 [dev-exemplar-BE.md](../examples/dev-exemplar-BE.md) 或 [dev-exemplar-FE.md](../examples/dev-exemplar-FE.md)（按端）；无状态机时 **四** 写「无」。

文首元数据（必填）：

```markdown
# [T-{id}] {任务名} — 构思落盘 [{BE|FE}]

- 任务：**T-{id}**（唯一）
- 端：**{BE|FE}**
- 功能：F-xxx · 契约：API-xxx（按需）
- 关联 REQ：REQ-xxx
- depends_on：T-xxx（无则写「无」）
- 写法对齐：`code-patterns-{端}.md` §x 或 `codebase/p1-{端}.md` §x
- 顺序：✅ 先构思后写码   （若先码后补：写 `顺序：⚠️ 先码后文档（无效构思，仅归档）` 且 **禁止勾 todo ①**）
```

## 七段 → ②③ 怎么用

| 段 | 章节 | 构思时必答 | ②③ 怎么用 |
|----|------|------------|-----------|
| 理解 | **一** | 核心目标、必须满足、**明确不做** | ② 不越界 |
| 数据 | **二** | 表/实体/字段/约束 | ② 建表、Entity |
| 契约 | **三** | BE：接口入出参；**FE+UI**：**UI↔API 映射表**（英文字段名）+ 调用清单 | ② 实现 API / 按表绑 UI |
| 状态 | **四** | 状态流转（无则写「无」） | ② 状态校验 |
| 流程 | **五** | **目的+需要什么+怎么做(5.x)**（BE→[exemplar-BE](../examples/dev-exemplar-BE.md)，FE→[exemplar-FE](../examples/dev-exemplar-FE.md)） | ② **按 5.x 做法**写码 |
| 边界 | **六** | 并发、幂等、异常场景 | ② 异常处理 |
| 依赖 | **七** | 框架、中间件、假设、humanTodo | ② 按栈实现 |
| 验收 | **八** | REQ AC + test/ac 规划名 | ③ 只跑此表 |
| 结果 | **九** | 表头 ① 必建 | ③ 回填 L3 |

**快速模式**：**一、五、八不可省**；二/三/四/六/七 写「无」或引用 REQ/model。W3 见 [flow-modes.md](flow-modes.md)。

## 铁律

| 铁律 | 说明 |
|------|------|
| **构思先于写码** | 无 dev → 禁止 Write 业务源码 |
| **一任务一构思** | 1 个 T-xxx = 1 个 dev 文件 |
| **七段齐全** | 一～七 + 八（九表头） |
| **五、核心流程逐步** | 禁止「实现某某模块」一句带过 |
| **顺序** | ① 构思落盘 → ② 按 **五** 写码 → ③ 按 **八** 验收 |

## 构思反模式

| ❌ 反模式 | ✅ 正确 |
|----------|---------|
| 口头想想直接写码 | Read exemplar → 完整模板 → 闸门 A → 勾① |
| 摘要 / 空壳九标题 | 五可执行（含类·方法痕迹） |
| 批量写完所有 dev 再写码 | 同一时刻只推进 1 个 T 的①→② |
| T≥3 自称快速 | 强制严谨 |
| 「见 REQ」代替五 | 写法步骤到 `Class.method` |

## 五、核心流程颗粒度（W3 检查）

| 要求 | 门槛 |
|------|------|
| **五** 结构 | **目的** + **需要什么**（表）+ **怎么做**（≥1 个 `5.x` 小节） |
| 每个 **5.x** | **要达成什么** + **需要什么** + **做法**（BE→类/方法；FE→页面/组件/方法） |
| 绑定 AC | 5.x 标题或验收表链 REQ 的 AC ID |
| 禁止 | 「实现 XX 模块」、无具体落点的空步骤 |
| 对照 | 与对应端 exemplar 同级或更细；**FE+UI** 时 **三、3.2** 每行可在 `API-xxx` 搜到字段名 |

省略条件（极少）→ [04-development.md](../phases/04-development.md)。
