# REQ 阶段 UI 设计描述模板

> **定位**：REQ 阶段描述**界面做什么、怎么交互**——供后续方案 `contracts/UI-xxx` 与 FE 开发生成 UI。  
> **不定义视觉样式**（配色、字体、圆角、动效细节等）——**样式由用户后续决定**（方案或开发前 AskQuestion / humanTodo）。  
> AC 规范：[ac-guide.md](ac-guide.md)

---

## 何时新建 UID 文档

| 条件 | 动作 |
|------|------|
| 有 Web / App / 小程序等**用户可见界面** | 评估是否需 UID |
| 页面/流程 ≥1 且交互非 trivial | **新建** `specs/requirements/ui/UID-xxx-名称.md` |
| 纯 API / CLI / 后台批处理、无 UI | **跳过**，REQ 注明「无 UI」 |
| 用户说「样式我自己定 / 先不定样式」 | 仍写 UID（结构与交互）；**样式节只写「待定」** |

**一页面/一主流程 ≈ 一份 UID**；复杂模块可拆多份，REQ 头部链全部 UID。

---

## 目录与命名

```
specs/requirements/
├── REQ-001-用户登录.md
├── ui/
│   ├── README.md              # UID 索引
│   ├── UID-001-登录页.md
│   └── UID-002-首页.md
└── README.md
```

| 前缀 | 含义 | 阶段 |
|------|------|------|
| **UID-xxx** | UI **设计描述**（结构、交互） | 阶段 1 REQ |
| **UI-xxx** | UI **技术契约**（路由、组件、API 绑定） | 阶段 3 sol contracts |

**禁止**在 REQ 阶段写 `solution/contracts/UI-xxx`——那是方案阶段产物。

## UID 文档模板（`UID-001-登录页.md`）

```markdown
# [UID-001] 登录页 — 界面描述

- 关联 REQ：[REQ-001](../REQ-001-用户登录.md)
- 平台：Web | 小程序 | App（与 REQ 一致）
- 状态：草稿 | 已确认
- **样式状态**：⬜ 待定（用户后续决定）

> **用途**：描述页面结构与交互，供生成 UI 实现参考。**不含**最终视觉样式。

---

## 1. 页面目标

- 用户在此完成：……
- 成功后跳转：……
- 失败时：……

---

## 2. 布局结构（信息架构，非视觉样式）

> 用区块/区域描述，**不写**具体颜色、字号、间距数值。

| 区域 | 内容 | 说明 |
|------|------|------|
| 顶栏 | Logo、标题 | 可选 |
| 主区 | 登录表单 | 见 UID「关键组件与交互」 |
| 底栏 | 协议链接 | 可选 |

**线框示意（可选，ASCII）**：

```
┌─────────────────────────┐
│         Logo            │
├─────────────────────────┤
│   [ 手机号输入框      ] │
│   [ 验证码输入框 [获取] ] │
│   [    登录按钮       ] │
├─────────────────────────┤
│   用户协议 | 隐私政策    │
└─────────────────────────┘
```

---

## 3. 关键组件与交互

| 组件 | 类型 | 交互行为 | 校验/限制 |
|------|------|----------|-----------|
| 手机号 | 输入框 | 11 位数字 | 非空、格式 |
| 验证码 | 输入框 + 按钮 | 点击「获取」→ 60s 倒计时 | 需先填手机号 |
| 登录 | 主按钮 | 提交 → loading → 成功/失败反馈 | 表单合法才可点 |

---

## 4. 状态与反馈

| 状态 | 用户看到什么 | 可执行操作 |
|------|--------------|------------|
| 初始 | 空表单 | 填写、获取验证码 |
| 提交中 | 按钮 loading | 不可重复提交 |
| 成功 | 跳转目标页 | — |
| 失败 | 错误提示（文案见 REQ AC） | 重试 |

## 6. 样式说明（REQ 阶段不定稿）

| 项 | REQ 阶段 | 谁决定 | 何时定稿 |
|----|----------|--------|----------|
| 配色 / 主题 | **待定** | **用户** | 方案 AskQuestion 或 humanTodo |
| 字体 / 字号 | **待定** | **用户** | 同上 |
```

## 阶段衔接

| 阶段 | 如何使用 UID |
|------|--------------|
| **1 REQ** | 新建/维护 UID；样式标「待定」 |
| **3 sol** | 读 UID → 生成 `contracts/UI-xxx`（路由、组件树、API 绑定）；AskQuestion **样式/设计系统** |
| **4 dev FE** | dev **二**（组件/状态）链 UID + UI-xxx；② 写码以用户确认的样式为准 |

## 反模式

| ❌ | ✅ |
|----|-----|
| REQ 里写 `#FF5733`、具体 px 间距 | 写「主按钮、表单区」，样式待定 |
| REQ 直接写 `contracts/UI-001` | 写 `requirements/ui/UID-001` |
| 有页面却不建 UID | 评估后建 UID 或 REQ 注明「无 UI」 |
| Agent 替用户定配色/字体 | 标待定 + humanTodo 或 sol AskQuestion |
