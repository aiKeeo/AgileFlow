## 规则（构思先行）

| # | 说明 |
|---|------|
| R1 | 无合规 dev → 禁止 Write 业务源码 |
| R2 | 构思必须落盘；聊天不算 |
| R3 | 1 T = 1 文件；禁止多 T 糅合 / 合并勾选 |
| R4 | **严谨**一～九标题+正文；**快速**一/五/八/九（+FE 三）；禁止摘要/空壳 |
| R5 | 五 = 目的→需要什么→5.x（做法含类·方法或页面路径） |
| R6–R8 | ② 只执行五；③ 只验八；催进度不豁免 |
| R9 | FE+UI：须有字面量 `### 3.1` `### 3.2` `### 3.3`；**3.1 含 ASCII 布局线条图**（对齐 UID，禁纯表）；3.2 英文字段 |
| R10 | **默认串行**：一次只推进 1 个 T；禁止批量写完所有①再写码。**例外**：用户显式并行 **且** 已过 [parallel 开闸卡](parallel-orchestration.md#并行开闸卡强制) **且** 走批次 B；单次批次 B **最多 3 个 T** 的①；仍禁止无并行许可的批量① |
| R11 | 连续做 ≠ 压缩模板 |
| R12 | 补盘/摘要/空壳 → 不勾 ①；父任务不得洗成正式 ✅；**即使用户说「接受技术债」也禁止**把「开发实现」标 ✅（只允许标 `⚠️ 补盘归档`，须另开正式①重做才能 ✅） |
| R13 | 闸门 A：段标题 + 五可执行 + **A6 反假九段** + **A7 机械 grep**；有文件但 grep 不过 → **禁止勾①** |
| R14 | `todo` 中 **T≥3** 或同时有 **BE+FE** → **强制严谨**（见 flow-modes） |
| R15 | 「全部开发」≠ 并行；**禁止**无显式并行就启 Task/Subagent 写业务源码；1 Task=1 T 且须已有合规① |
| R16 | 标「开发实现 ✅」前须**同时**满足：① 合法 T 头（`^#{3,4} T-\d+`，**仅 ###/####**）≥1；② 每个 T 头下有 ①②③ 三行（含 `atlas/dev/T-xxx` 路径）；③ `count(atlas/dev/T-*.md)`（非⚠️、非 README、非 temp/）= 合法 T 头数；④ 每 T 闸门 C 证据齐。扁平列表或有头无三段式 → **格式违规禁止 ✅** |
| R17 | 进阶段 4 /「全部开发」：**先 TodoWrite 展开**——每个 T **必须三条**「①构思 / ②写码 / ③AC」；禁止一条 Todo 多 T；未展开禁止 Write 业务源码（见 [todo.md](../templates/todo.md#todowrite-强制展开防漏①--最高优先级)） |
| R18 | **闸门 C（可运行）**：每 T 勾③前必须 **编译 + 能启/能调接口 + 本 T 冒烟**；F-xxx/MVP 切片完再跑模块冒烟；「给用户看」前存在端启动冒烟全过；证据写入 dev 九；单测绿 ≠ 可交（见 [dev-quickstart 闸门 C](../templates/dev-quickstart.md#闸门-c可运行每-t--每模块强制)） |

催进度反模式 → [SKILL.md](../SKILL.md)。Subagent 细则 → [parallel-orchestration.md](parallel-orchestration.md)。

## 严谨 vs「全部做 / 直接全开发」

| 模式 | 「全部做 / 连续做 / 直接全开发」时 |
|------|-----------------------------------|
| **快速** | 仅未命中强制严谨时；任务间连续 **完整** ①→②→**C**→③；仍过闸门 A/B/C |
| **严谨** | 不适用连进压缩；逐 T **完整一～九**；可连续，不可摘要/空壳；仍过闸门 C |

**客观强制严谨**（命中任一 → 必须严谨，用户说「快速」也拒绝）：

- `atlas/todo.md` 开发任务 **≥3 个** T 头（行匹配 `^#{3,4} T-\d+`，仅 `###`/`####`）
- 或同时存在 `[BE]` 与 `[FE]` 任务
- 或支付/权限/DB·服务端/用户数据/多模块/>1 天

## 三段闸门（A 构思 / B 写码 / C 可运行）

```
Read exemplar → 写 1 个 T 的 dev →【闸门 A】→ 勾 ① →【闸门 B】→ ② →【闸门 C 可运行】→ ③
（禁止：先写完 T-002～T-011 的 dev 再统一写码；禁止跳过闸门 C）
```

### 闸门 A — 勾 ① 前

| # | 过线标准 | 不过 → |
|---|----------|--------|
| **A0** | 已 Read 本端 exemplar（BE→exemplar-BE / FE→exemplar-FE） | 先 Read |
| **A1** | **默认串行**：仅 1 个进行中 T；无其他 T「①已勾且②未完成」。**例外（显式并行）**：开闸卡已确认且走 [parallel](parallel-orchestration.md)——**批次 B** 允许多个 T「①已勾、②未开」（**≤3**）；**批次 C** 允许多个 T **同时**处于②/③（1 Task=1 T）。**仍禁止**：无开闸时出现多份「①勾完②未完」；「全部开发」冒充并行 | 串行：先完成或回滚；伪并行：改串行 |
| **A2** | `atlas/dev/T-xxx-*.md` 存在 | 先写文件 |
| **A3** | **段标题**过线（见下） | 补标题 |
| **A4** | **五可执行**过线（见下）；FE+UI 另过 **3.x 字面量** | 重写五 / 补 3.1–3.3 |
| **A5** | 非补盘/非摘要；无 `顺序：⚠️` | 标 ⚠️；重写 |
| **A6** | **反假九段**过线（见下） | 按 exemplar 重写整份 |
| **A7** | **机械 grep**全过（见下）；须有命中证据 | 补字面量后重 grep；**禁止勾①** |

#### A3 段标题（可 grep）

| 模式 | 须存在 |
|------|--------|
| **严谨** | `## 一、` … `## 九、` **全部**（字面量含 `## `） |
| **快速** | `## 一、` `## 五、` `## 八、` `## 九、`；FE+UI 另须 `## 三、`；**二/四/六/七 标题可省略**（省略则不写该段） |

五内须有：`### 目的` `### 需要什么` `### 怎么做` ≥1× `#### 5.`

**禁止**：用纯文本 `一、` `二、`…（无 `##`）冒充段标题——这是 **假九段**，A3/A6 不过。

#### A4 五可执行（防空壳标题）

每个 `#### 5.` 小节须**同时**满足：

1. 含字面量 **要达成什么** / **需要什么** / **做法**（或同等加粗标题）
2. **做法**下 ≥2 条编号步骤
3. 做法中 ≥1 处反引号代码痕迹：`` `ClassName.method` `` 或 `` `pages/...` `` 或 `` `services/...` ``
4. **禁止**五的正文（目的/需要什么/做法）整段仅为「见 REQ」「见 API」「见 contracts」「实现即可」
5. **禁止**用「五、关键代码落点：Xxx.java」一句话代替整个五（无 `### 目的` / `#### 5.` = 不过）

FE+UI 另须可 grep：`### 3.1` `### 3.2` `### 3.3`；3.2 表含英文契约字段列；**3.1 正文须含布局线条图**（代码块内有 `┌`/`│`/`+--` 之一，且非空）。

#### A6 反假九段（bbb 事故形态 · 机械）

勾①前 Agent **必须自检**（任一项命中 → **禁止勾①**，重写）：

| # | 命中即失败 |
|---|------------|
| F1 | 全文几乎无 `## 一、` / `## 五、`（只有 `一、`/`五、` 纯文本） |
| F2 | 无 `### 目的` 或无 `#### 5.` |
| F3 | 「五」无 `` `Class.` `` / `` `pages/` `` / `` `services/` `` 痕迹 |
| F4 | FE+UI 无 `### 3.1` 或 3.1 无线条图字符 |
| F5 | 文中出现「② 写码完成 / 已写码 / 等待联调」却仍要勾**正式①**（属补盘或事后摘要） |
| F6 | 全文过短：去掉空白后少于约 800 字符，且无 ≥2 个 `#### 5.`（纯脚手架可 1 个 5.x，但仍须有完整五结构，不得假九段） |

**金标准**：与 exemplar 同结构；颗粒度可略简，**结构不可降级成摘要**。

#### A7 机械 grep（勾①前强制 · 假构思勾不了）

> **不靠记住。** 勾①前必须对本 `atlas/dev/T-xxx-*.md` **实际 Grep**；缺字面量或命中禁形 → **禁止** TodoWrite①=`completed` / todo①=`[x]`。  
> 速查表 → [dev-quickstart A7](../templates/dev-quickstart.md#a7-机械-grep勾①前强制--假构思勾不了)。

**必过字面量**：`## 一、` · `## 五、` · `### 目的` · `### 需要什么` · `### 怎么做` · `#### 5.` ·（`` `Class.` `` 或 `pages/` 或 `services/`）  
**FE+UI 另加**：`### 3.1`（含 `┌`/`│`/`+--`）· `### 3.2` · `### 3.3`  

**禁形（ai-error-case）**：`## 一、目标` · `## 五、可执行方案` · `## ① 构思` / `## ② 关键实现点` —— 出现且缺「目的/5.」结构 → 不过。

勾①时回复须含一句：`A7 grep：G1–G7[ +G8/G9] 已命中`（可附路径）。无此句视为未过 A。

### 闸门 B — Write 源码前

| # | 过线标准 |
|---|----------|
| B1 | 该 T 的 ① 已勾 |
| B2 | 仍只做一个 T；Write ⊆ 五当前 5.x |
| B3 | 已 Read **本端**写法锚点（`p1-frontend|backend` 或 `code-patterns-*`）——**至少含资产索引**；严谨另对齐 §三 |
| B4 | 有 UI/API 时：复用盘点已填；表写「复用」则 5.x/代码须用该路径，禁止平行实现 |
| B5 | **严谨**：七含具体 `资产索引`行 + `§3.x`；**快速**：B3+B4 即可（七可无） |

### 闸门 C — 勾 ③ / 父任务 ✅ 前（可运行）

| # | 过线标准 | 不过 → |
|---|----------|--------|
| **C1** | 本 T 涉及端 **编译/构建**成功（L1+L2） | 修编译，禁止勾③ |
| **C2** | 按 [C2 分档](../templates/dev-quickstart.md#c2-分档堵单测绿但起不来--不改流程)：**该端第 1 个 T 须真启动**；后续单 T 可 MockMvc；**F/MVP 切片与给用户看须真实启动** | 修启动与配置 |
| **C3** | 本 T 主路径冒烟不炸（不 500；核心读写走通） | 修功能 |
| **C4** | 证据已写入 dev **九** 或 todo（命令+结果） | 补证据 |

权威细则与「给用户看 / 模块级」时机 → [dev-quickstart 闸门 C](../templates/dev-quickstart.md#闸门-c可运行每-t--每模块强制)。  
阶段 5 的 5-0 **不能替代**本闸门。

## ①②③ 要点

- **①** 模板 → [dev-rationale](../templates/dev-rationale.md)；照 exemplar 颗粒度  
- **②** 只把五变成代码；FE 键名对齐 3.2  
- **③** 先过 **闸门 C**，再只测八；九回填（含 C 证据 + L3）；父任务 ✅  

### 默认串行（禁止批量① / 禁止 Subagent 外包）

0. **TodoWrite 展开**（R17）：为每个 T 建 **①②③ 三条**（强制）；「全部开发」时①条目须先全部出现为 pending  
1. 取**一个** ①未勾 的 T → 进行中  
2. A0→写该 T 的 dev→闸门 A→勾①（TodoWrite + atlas/todo）→闸门 B→②→**闸门 C**→③  
3. **在该 T 的 ② 开始前，禁止**再为其他 T 写 dev 或勾①（**显式并行批次 B 除外**，见下）  
4. 「全部开发」时重复 1–3，**仍由主 Agent 串行**；禁止用 1～2 个 Subagent「实现全部 BE/FE」；**禁止跳过闸门 C**  
5. 全部父任务 ✅ **且** `dev` 文件数 = T 头数 **且** 每个 T 的 TodoWrite① completed **且** 闸门 C 证据齐 → 结束闸门 → 停  
6. 任一 **F-xxx / MVP 切片** 相关 T 全 ✅ → 模块编译+冒烟（R18）→ **AskQuestion 阶段性确认卡** → **停止**（见 [askquestion-gate](../templates/askquestion-gate.md#阶段性确认卡阶段-4-内--mvp--f-xxx-切片强制)）  
7. 用户要「给用户看」或选「先给我看」→ 存在端全量启动冒烟过线 + 给出启动步骤；等用户反馈  
8. 全部 T ✅ → 阶段闸门（进 tests）→ 停；prompt 须含闸门 C 证据摘要 

### 显式并行时（与 A1/R10 的关系）

仅用户原话含「并行 / subagent / 同时开 FE+BE」时启用 → [parallel-orchestration.md](parallel-orchestration.md)：

| 批次 | 允许 | 仍禁止 |
|------|------|--------|
| B（齐①） | 多个 T 先各自写合规 dev、过 A、勾① | 「全部开发」冒充并行；无①就② |
| C（并行②） | 1 Task = 1 T，且该 T ①已勾 | 一个 Subagent 多 T；Subagent 写①或改 todo ✅ |

**① 只由主 Agent 写**；Subagent **只做②→闸门C→③**。
