- 预计完成时间：

## 流程进度

- [ ] 需求澄清
- [ ] 数据建模
- [ ] 方案设计（sol:）
- [ ] 开发实现
- [ ] 测试验收

> **快速 / 严谨 / AI 自主均须维护本区勾选。**  
> 快速 ≠ 跳过上列阶段；AI 自主 ≠ 不写进度。建模按需跳过时标 `⏭️` 并写依据。

## 决策委派（可选）

> 见 [stage-delegation.md](../templates/stage-delegation.md)  
> **「用户不用管 / 后面都你定」写这里（AI自主），不要当成可以薄化下方开发任务。**

- 全局：未设置 | 用户决策 | **AI自主**
- 自阶段：— （例：`req` 起 AI自主）
- 备注：用户说「这阶段我来」→ 仅当阶段改回用户决策

## 开发任务

> **每项必须拆成三步子项**；**① 未勾 → 禁止 ② 写码**。  
> **`atlas/todo.md` 三段式强制**（快速/严谨/AI自主均同）：每个任务 = `### T-xxx` + ①②③；**不是推荐、不可省略**。  
> **快速模式同样详细**：禁止因「快速」合并多 T、省略 ①②③、或写成「实现 MVP」一条。  
> **阶段 4 入口铁律**：必须用 **`TodoWrite` 为每一个 T 各建三条**（①构思 / ②写码 / ③AC）；**禁止**一条 Todo 覆盖多个 T；**禁止**只建①。  
> 父任务 `- [x]` = 三步全 ✅；全部父任务 ✅ → **AskQuestion 阶段闸门**。  
> 任务头格式：`### T-xxx：[端] 描述（估时）— 功能ID [/ 契约ID]`（**禁止**写成 `- [ ] T-xxx 描述` 一行）  
> 端标记：`[BE]` 后端 | `[FE]` 前端 | `[FULL]` 不可拆分

### ① 质量门槛（勾①前 · **冻结区** · 阶段 3 写入后全程有效）

> **有文件 ≠ 过闸门 A。** 对照 [exemplar-BE](../examples/dev-exemplar-BE.md) / [exemplar-FE](../examples/dev-exemplar-FE.md)。  
> 细则 → [闸门 A](dev-quickstart.md#闸门-a勾-①-前) · **[A7 机械 grep](dev-quickstart.md#a7-机械-grep勾①前强制--假构思勾不了)**（假构思勾不了）。  
> **冻结**：本节 Q1–Q6 +「机械 grep」表由 skill 规定；**禁止**改成项目私货（XSS/JDK/业务规则等写 `humanTodo` 或 REQ）。改写门槛 = **无效**，勾①仍须按原表 + A7。

| # | 必须 | 禁止（假构思 · 不过 A） |
|---|------|------------------------|
| Q1 | 已 **Read** 本端 exemplar | 凭记忆写短摘要 |
| Q2 | 段标题用 Markdown：`## 一、` `## 五、`…（快速可省二四六七） | 纯文本 `一、二、三…` 无 `##`（**假九段**）；`## 一、目标` / `## 五、可执行方案` 薄稿 |
| Q3 | 五内有 `### 目的` `### 需要什么` `### 怎么做` + ≥1× `#### 5.` | 五只有「关键代码落点」或编号文件列表 |
| Q4 | 每个 `#### 5.`：要达成什么/需要什么/做法≥2步 + `` `Class.method` `` 或 `pages/` | 「见 REQ / 实现即可」 |
| Q5 | FE+UI：`### 3.1`（含 ASCII 线条图）+ `### 3.2`（英文字段）+ `### 3.3` | 只有「视图：logo+表单」无图无映射表 |
| Q6 | ① 阶段禁止写「②已写码/写码完成」装构思 | 事后补盘却勾正式① |
| **Q7** | **勾①前对本文件跑 A7 机械 grep，全过并留证据** | 未 grep / 缺字面量仍 `[x]` |

### 机械 grep（勾①卡死 · 与 A7 同）

## 状态标记

| 标记 | 含义 |
|------|------|
| 三步子项 `- [ ]` | 该步未开始/进行中 |
| 三步子项 `- [x]` | 该步已完成 |
| 父任务 `- [x]` | ①②③ 三步全 ✅ |
| 「进行中」区 | 当前正在做的 **T-xxx**（含当前步 ①/②/③） |

## TodoWrite 强制展开（防漏① · 最高优先级）

> **记不住规范 → 靠清单。** 阶段 4 / 「全部开发」时，Agent **必须先**把每个 T 的构思步骤写进 **TodoWrite**，再一个个做。  
> `atlas/todo.md` 与 **TodoWrite** 必须同步；**以 TodoWrite 可见条目为准推进**。

### 何时必须展开

命中任一 → **本回复内先 TodoWrite，再写任何业务源码**：

1. 进入阶段 4  
2. 用户说「全部开发 / 直接全开发 / yes_all / 连续做」  
3. 从暂停恢复继续开发  

### 必须建成的条目（每个 T 三条①②③）

| 要求 | 说明 |
|------|------|
| **1 T = 3 条 TodoWrite（强制）** | 每个 T 头 **必须**有 TodoWrite：`① 构思→atlas/dev/T-xxx…` + `② 写码 T-xxx` + `③ AC T-xxx`；**禁止**只建①不建②③ |
| **① 条目 content 必含** | 字面量 `①` + `构思` + 路径 `atlas/dev/T-xxx` + **`exemplar` 或 `闸门A` 或 `A7` 或 `grep`**（提醒对照金标准+机械检查） |
| **id 命名** | `T-002-1` / `T-002-2` / `T-002-3`（或 `t002-dev`）；**禁止** `be-all` / `t002-018` |
| **顺序** | 未完成的 `T-xxx-①` 全部做完之前，**禁止**把任意 `T-yyy-②` 标 completed（显式并行批次 B 齐①除外，见 parallel；单批最多 3 T） |
| **勾 completed 条件（①）** | ① 文件存在（非 README/temp）；② **闸门 A 全过（含 A6+A7）**；③ **已对本文件 Grep** 且 G1–G7（FE+UI 含 G8/G9）命中；④ 回复含 `A7 grep：…已命中`。缺任一 → **禁止** completed |
| **勾 completed 条件（②）** | 该 T 的①已 completed；Write ⊆ 五 |
| **勾 completed 条件（③）** | 该 T 的②已 completed；**闸门 C 已过**（编译+能启/接口+冒烟，证据已写）；八中可自动化 AC 已绿 |

### ❌ 禁止（洗清单）

```
❌ TodoWrite 一条：「实现 T-002~T-018」/「全部 API」/「后端+前端」
❌ 只建 T-001 三条，其余 T 不建①就写码
❌ 把多个 T 的①合并成「写完所有 dev」一条
❌ ① 未建文件就标 completed
❌ ① 有文件但未 A7 grep / 是「一、目标+五、可执行方案」薄稿仍 completed
❌ 把 todo「①质量门槛」改成项目私货当过关依据
❌ 「全部开发」时跳过 TodoWrite 直接开写
❌ atlas/todo 写成扁平列表：`- [x] T-001 脚手架`（无 ### T 头、无 ①②③）
❌ 只用 `atlas/dev/README.md` 索引代替每个 `atlas/dev/T-xxx-*.md`
```

### ✅ 正确（18 个 T 时）

```
TodoWrite（可分批 merge，但①必须先全部出现为 pending）:
  T-001-1 | ① 构思 → atlas/dev/T-001-…-BE.md（对照 exemplar / 闸门A）  pending
  T-001-2 | ② 写码 T-001                          pending
  T-001-3 | ③ AC T-001                            pending
  T-002-1 | ① 构思 → atlas/dev/T-002-…-BE.md（对照 exemplar / 闸门A）  pending
  …（每个 T 都有 ①②③ 三条）
```

「全部开发」= **按 TodoWrite 从上到下**做完每一条；**不是**跳过清单出代码。

## 规则

- 开发任务只维护此处；**一任务 = 一个 `### T-xxx`（或 `####`）头 + ①②③ 三步子项**（**强制**，非推荐）
- 阶段 3：拆解任务 + 功能依赖；**每个 T-xxx 须预填三步子项与 dev 路径**；不过 [S1–S4](solution-architecture.md#阶段-3-结束前自检不过线禁止标方案已确认--禁止进阶段-4) 禁止确认方案
- 阶段 4：**先 TodoWrite 展开①②③** → 再严格按子项顺序 ①→②→闸门C→③（细则 [04-development.md](../phases/04-development.md)）
- 阶段 4 开始 **T-xxx** 时：**必须 `TodoWrite` 同步**；会话内与 `atlas/todo.md` 同步勾选
- **禁止**：跳过 ① 直接写码；三步未全 ✅ 就把父任务标完成
- **禁止扁平列表**：`- [ ] T-001 描述` 一行完事（无三段式）= 阶段 3/4 违规
- **禁止合并**：不得写成 `### T-002 ~ T-013 ✅` 或 TodoWrite 一条「实现 T-002~013」；**一次只推进 1 个 T-xxx**（显式并行：可先齐多 T 的①，②仍 1 Task=1 T）
- **勾 ① 前**：对应 `atlas/dev/T-xxx-*.md` 必须已存在且过闸门 A（禁止「先勾再补文件」；**`dev/README.md` 不算①**）
- **标「开发实现 ✅」前自检**：
  1. 合法 T 头数（`^#{3,4} T-\d+`，**仅 ###/####**；`## T-` **不算**）≥1；若开发任务区出现 `T-\d+` 但合法 T 头=0 → **格式违规，禁止 ✅**
  2. **每个**合法 T 头下须有含字面量 `①` `②` `③` 的三行子项（① 行含 `atlas/dev/T-xxx`）；缺任一行 → **禁止 ✅**
  3. `atlas/dev/T-*.md` 个数（排除文首 `顺序：⚠️` 补盘；**排除 README / temp/**）**=** 合法 T 头数
  4. TodoWrite 中每个 T 的①②③均为 completed；每 T 闸门 C 有证据
- **补盘**：先码后写的 dev 文首标 `顺序：⚠️ 先码后文档`；**禁止勾 ①**；父任务只可标 `⚠️ 补盘归档`  
  - **禁止**把「开发实现」洗成 ✅——**即使用户说「接受技术债 / 先这样」也不行**  
  - 要正式 ✅：必须**另写合规①**（无 ⚠️）→ 过闸门 A → 再按 ②→C→③ 重验（可复用已有源码，但流程不可跳）
- **勾 ③ 前**：须过 [闸门 C](dev-quickstart.md#闸门-c可运行每-t--每模块强制)；禁止「单测绿但起不来」仍 ✅
- **「给用户看 / MVP」**：存在端编译+启动+主路径冒烟全过，才允许说可看
- **「全部开发」**：主 Agent 串行；先展开 TodoWrite；每 T 含闸门 C；禁止 Subagent 一次写完多 T
