# 项目待办事项

- 项目名称：
- 创建时间：
- 预计完成时间：

## 流程进度
- [ ] 项目盘点（init，仅 brownfield）
- [ ] 需求澄清
- [ ] 数据建模
- [ ] 方案设计（sol:）
- [ ] 开发实现

## 决策委派（可选）

> 见 [stage-delegation.md](../templates/stage-delegation.md)

- 全局：未设置 | 用户决策 | **AI自主**

## 开发任务

> **每项必须拆成三步子项**；**① 未勾 → 禁止 ② 写码**。  
> **阶段 4 入口铁律**：必须用 **`TodoWrite` 为每一个 T 各建独立条目**（至少每 T 一条「① 构思」；推荐每 T 三条 ①②③），**禁止**一条 Todo 覆盖多个 T。  
> 父任务 `- [x]` = 三步全 ✅；全部父任务 ✅ → **AskQuestion 阶段闸门**。  
> 任务头格式：`任务ID：[端] 描述（估时）— 功能ID [/ 契约ID]`  
> 端标记：`[BE]` 后端 | `[FE]` 前端 | `[FULL]` 不可拆分

### T-001：[BE] {后端任务简述}（{估时}）— F-001

## 状态标记

| 标记 | 含义 |
|------|------|
| 三步子项 `- [ ]` | 该步未开始/进行中 |
| 三步子项 `- [x]` | 该步已完成 |
| 父任务 `- [x]` | ①②③ 三步全 ✅ |
| 「进行中」区 | 当前正在做的 **T-xxx**（含当前步 ①/②/③） |

---

## TodoWrite 强制展开（防漏① · 最高优先级）

> **记不住规范 → 靠清单。** 阶段 4 / 「全部开发」时，Agent **必须先**把每个 T 的构思步骤写进 **TodoWrite**，再一个个做。  
> `atlas/todo.md` 与 **TodoWrite** 必须同步；**以 TodoWrite 可见条目为准推进**。

### 何时必须展开

命中任一 → **本回复内先 TodoWrite，再写任何业务源码**：

1. 进入阶段 4  
2. 用户说「全部开发 / 直接全开发 / yes_all / 连续做」  
3. 从暂停恢复继续开发  

### 必须建成的条目（每个 T 至少 1 条①）

| 要求 | 说明 |
|------|------|
| **1 T ≥ 1 条 Todo** | 每个 T 头对应 ≥1 条 TodoWrite；**推荐**每 T 三条：`T-xxx-①` / `T-xxx-②` / `T-xxx-③` |
| **① 条目 content 必含** | 字面量 `①` + `构思` 或 `dev` + 路径 `atlas/dev/T-xxx` |
| **id 命名** | `T-002-1` / `T-002-2` / `T-002-3`（或 `t002-dev`）；**禁止** `be-all` / `t002-018` |
| **顺序** | 未完成的 `T-xxx-①` 全部做完之前，**禁止**把任意 `T-yyy-②` 标 completed（显式并行批次 B 齐①除外，见 parallel） |
| **勾 completed 条件（①）** | `atlas/dev/T-xxx-*.md` **已存在** 且闸门 A 过线；禁止空勾 |
| **勾 completed 条件（②）** | 该 T 的①已 completed；Write ⊆ 五 |
| **勾 completed 条件（③）** | 该 T 的②已 completed；八中可自动化 AC 已绿 |

### ❌ 禁止（洗清单）

```
❌ TodoWrite 一条：「实现 T-002~T-018」/「全部 API」/「后端+前端」
❌ 只建 T-001 三条，其余 T 不建①就写码
❌ 把多个 T 的①合并成「写完所有 dev」一条
❌ ① 未建文件就标 completed
❌ 「全部开发」时跳过 TodoWrite 直接开写
```

### ✅ 正确（18 个 T 时）

```
TodoWrite（可分批 merge，但①必须先全部出现为 pending）:
  T-001-1 | ① 构思 → atlas/dev/T-001-…-BE.md     pending
  T-001-2 | ② 写码 T-001                          pending
  T-001-3 | ③ AC T-001                            pending
  T-002-1 | ① 构思 → atlas/dev/T-002-…-BE.md     pending
  …（每个 T 都有 ①；推荐②③也建）
```

「全部开发」= **按 TodoWrite 从上到下**做完每一条；**不是**跳过清单出代码。

## 规则

- 开发任务只维护此处；**一任务 = 一个 T 头 + 三步子项**；T 头推荐 `### T-xxx`（挂在 MVP 分组下可用 `#### T-xxx`，计数时 `###`/`####` 都算）
- 阶段 3：拆解任务 + 功能依赖；**每个 T-xxx 须预填三步子项与 dev 路径**
- 阶段 4：**先 TodoWrite 展开** → 再严格按子项顺序 ①→②→③（细则 [04-development.md](../phases/04-development.md)）
- 阶段 4 开始 **T-xxx** 时：**必须 `TodoWrite` 同步**；会话内与 `atlas/todo.md` 同步勾选
- **禁止**：跳过 ① 直接写码；三步未全 ✅ 就把父任务标完成
- **禁止合并**：不得写成 `### T-002 ~ T-013 ✅` 或 TodoWrite 一条「实现 T-002~013」；**一次只推进 1 个 T-xxx**（显式并行：可先齐多 T 的①，②仍 1 Task=1 T）
- **勾 ① 前**：对应 `atlas/dev/T-xxx-*.md` 必须已存在且过闸门 A（禁止「先勾再补文件」）
- **标「开发实现 ✅」前自检**：`atlas/dev/T-*.md` 个数（排除文首含 `顺序：⚠️` 的补盘）**=** 本文件 T 头个数（`^#{2,4} T-\d+`）；且 TodoWrite 中每个 T 的①均为 completed
- **补盘**：先码后写的 dev 文首标 `顺序：⚠️ 先码后文档`；**禁止勾 ①**；父任务标 `⚠️ 补盘归档`，**禁止**把「开发实现」洗成 ✅（除非用户显式接受技术债）
- **「全部开发」**：主 Agent 串行；先展开 TodoWrite；禁止 Subagent 一次写完多 T
