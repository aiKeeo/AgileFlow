## 序（不可跳、不可批量①、不可 Subagent 外包、不可交不能跑的码）

```
【默认·主 Agent 串行】
0. 确认 atlas/todo 已是 ### T-xxx + ①②③（扁平 `- [ ] T-001` 须先改写，禁止直接写码）
1. TodoWrite 展开：每个 T **三条**「①构思 / ②写码 / ③AC」
   ※ 未展开 → 禁止 Write 业务源码（见 todo.md「TodoWrite 强制展开」）
2. 取下一条未完成的① → Read exemplar → 写【该】T 完整 `atlas/dev/T-xxx-*.md`
   →【构思闸门】→ 勾 ①（TodoWrite + atlas/todo.md）→【写码闸门】→ ②
   →【可运行闸门 可运行】→ ③
3. 下一条①再重复。禁止先写完多份 dev 再统一写码（无并行许可时）。
   ※ `dev/README.md` 索引 ≠ ①；每个 T 必须独立文件
```

**连续做 / 全部开发 / yes_all** = **先展开 TodoWrite**，再按清单逐条做（每条含可运行闸门）；**≠** 摘要 / 空壳 / 合并多 T / **从不编译启动** / **跳过 TodoWrite** / **扁平 todo**。

| 用户说 | 主 Agent 必须 | 禁止 |
|--------|---------------|------|
| 全部开发 / 直接全开发 | **先**确认三段式 todo → TodoWrite 为每个 T 建① → 串行 ①→②→可运行闸门→③ | 启 Task 批量写码；扁平 `- [ ] T-001`；1 个 Todo 多 T；无①就写码；跳过可运行闸门；只用 README 假构思 |
| 并行 / 多 subagent（显式） | 先读 [parallel-orchestration](../phases/parallel-orchestration.md)；每 T 先有合规①；②后仍过可运行闸门 | 无 dev 就让 Subagent Write；Subagent 跳过 C |
| 给用户看 / MVP 演示 | 存在端编译+启动+MVP 主路径冒烟全过 | 「代码齐了」就甩给用户 |

**未读到用户原话含「并行/subagent」→ 禁止启用「串行约束」并行例外。**

**标「开发实现 ✅」前**：开发完成格式门槛 全过——合法 T 头（`^#{3,4} T-\d+`）≥1 **且** 每头下有①②③ **且** `dev/T-*.md` 数 = 头数（**README/temp 不算**）**且** TodoWrite①②③ completed **且** 每 T 可运行闸门 有终端证据。  
若开发任务区有 `T-\d+` 但合法 T 头=0（扁平列表）或有头无三段式 → **禁止 ✅**。

## 构思闸门（勾 ① 前）

| 检查项 | 标准 |
|--------|------|
| **读金标准** | 已 Read exemplar-BE 或 exemplar-FE |
| **串行约束** | **串行**：只 1 个 T；无其他「①勾完②未完」。**显式并行**：构思齐批 允许多个①已勾②未开；写码执行批 允许多 T 同时②（1 Task=1 T） |
| **文件存在** | `atlas/dev/T-xxx-*.md` 已落盘 |
| **九段标题** | 段标题须为 `## 一、` 等（**禁止**纯文本 `一、` 假九段） |
| **五可执行** | `### 目的` + `#### 5.` + Class.method/pages；FE 则 3.1/3.2/3.3（**3.1 含线条图**） |
| **非补盘** | 非 ⚠️ 补盘 / 非摘要 |
| **反假九段** | 见下「反假九段」表（纯文本假标题 / 过短薄稿等） |
| **字面量校验** | **机械 grep 全过**（见下）——**不过禁止勾①**；有文件≠过构思闸门 |
| **脚本校验** | **须跑** `--gate dev-step1-literal --dev-file atlas/dev/T-xxx-*.md` **exit 0**（命令用 `--print-cmd` 或见 [validate-atlas-gate](validate-atlas-gate.md)）；回复贴 `字面量校验：…已命中` + 摘要 |

### 字面量校验（勾①前强制 · 假构思勾不了）

> **不靠记住，靠卡死。** 勾 TodoWrite①=`completed` 或 `atlas/todo` ①=`[x]` **之前**，必须对本文件跑 Grep（或等价），**全部必过项命中**；回复中须贴出命中证据（路径+关键行）。  
> 权威细则亦可对照 [04 字面量校验](../phases/04-development.md#字面量校验勾①前强制--假构思勾不了)。

**必过（BE/FE 通用）**——缺任一 → **禁止勾①**：

| 检查项 | 文件中须存在字面量 |
|--------|-------------------|
| **一段标题** | `## 一、` |
| **五段标题** | `## 五、` |
| **目的小节** | `### 目的` |
| **需要什么** | `### 需要什么` |
| **怎么做** | `### 怎么做` |
| **5.x 步骤** | `#### 5.` |
| **代码落点** | `` ` `` 内含 `.` 的类方法 **或** `pages/` **或** `services/`（至少一处） |

**FE+UI 另必过**：

| 检查项 | 须存在 |
|--------|--------|
| **FE 布局图** | `### 3.1` 且正文含 `┌` / `│` / `+--` 之一 |
| **FE 字段表** | `### 3.2` 与 `### 3.3` |

**命中即失败（ai-error-case 薄稿形态）**——出现且缺「目的小节～5.x 步骤」→ **禁止勾①**：

| 禁形 | 说明 |
|------|------|
| `## 一、目标` | 不是模板「一、需求理解」 |
| `## 五、可执行方案` | 不是「五、核心流程」+ `### 目的`/`#### 5.` |
| `## ① 构思` / `## ② 关键实现点` | 用①②③当章节冒充九段 |
| `## 八、验证` 且无 `## 八、REQ` 类验收表头、且全文无 `### 目的` | 薄验证段冒充八 |

**推荐 Grep（对本 T 文件）**：

```
pattern: ## 一、|## 五、|### 目的|### 需要什么|### 怎么做|#### 5\.|### 3\.1|### 3\.2|### 3\.3
```

**❌ 不过（bbb / ai-error-case）**

```
❌ 假九段：一、…二、…五、关键代码落点：AuthService.java（无 ## / 无 #### 5.）
❌ ## 一、目标 + ## 五、可执行方案 编号列表（无 ### 目的 / #### 5.）→ 仍勾①
❌ 摘要十几行冒充构思
❌ 空壳九个 ## 标题无正文
❌ FE 无 3.1 线条图 / 无 3.2 英文字段表
❌ 文末「② 写码完成」却勾正式①
❌ T-001 写满、T-002 起偷懒成摘要
❌ 未跑字面量校验 / 无命中证据就 TodoWrite①=completed
```

### 段标题

- **严谨**（T≥3 或 BE+FE 或 DB/权限等 → 强制）：`## 一、`～`## 九、` 全有  
- **快速**：九段标题全齐；二/四/六/七 可 `→ 引用`+**待补齐**；一/五/八/九（FE+UI +`## 三、`）须实质；勾①前须跑 `--gate dev-step1-literal`

五内：`### 目的` `### 需要什么` `### 怎么做` `#### 5.`

### 五可执行（防空壳）

每个 `#### 5.`：

- 有 **要达成什么 / 需要什么 / 做法**
- 做法 ≥2 步，且含 `` `Class.method` `` 或 `` `pages/...` `` / `` `services/...` ``
- 禁止五的正文只有「见 REQ / 见 API / 实现即可」
- 禁止「五、关键代码落点：Xxx.java」一句话顶替五

FE+UI：须有 `### 3.1` `### 3.2` `### 3.3`；**3.1 须含 ASCII 布局线条图**（对齐 UID）

## 写码闸门（Write 前）

| 检查项 | 标准 |
|--------|------|
| **① 已勾** | 该 T 构思步骤已完成 |
| **单任务约束** | 仍只做一个 T；Write ⊆ 当前「五、核心流程」的 5.x |
| **读过写法锚点** | 已 Read **本端**锚点（至少**资产索引**；见 [code-conventions](code-conventions.md)） |
| **复用盘点就绪** | 有 UI/API 时复用盘点已填；表写「复用」则 5.x/代码须用该路径，禁止平行实现 |
| **严谨资产对齐** | **严谨**：七含具体资产行 + `§3.x`；**快速**：读过锚点 + 复用盘点即可 |

## 可运行闸门（每 T / 每模块强制）

> **写完码 ≠ 能用。** 勾 ③、父任务 ✅、或说「可以给用户看」之前，Agent **必须亲自跑终端**过线。  
> 命令以 `architecture.md` 为准；无约定用下表默认。

### 何时跑

| 时机 | 范围 |
|------|------|
| **每个 T** 在 ② 写完、勾 ③ 前 | 本 T 涉及的端 + 本 T 主路径 |
| **一个 F-xxx / MVP 切片** 相关 T 全 ✅ 时 | ① 该模块编译 + 模块冒烟（可运行闸门）→ ② **AskQuestion**（含是否 Playwright FE 冒烟，见 [fe-smoke-playwright](fe-smoke-playwright.md)）→ 阶段性确认卡 → 停 |
| **给用户看 / MVP 演示 / 交付体验** | 存在端全量：编译 + 启动探针 + MVP 主路径冒烟 → **可询问** FE Playwright → 再问用户或给出启动步骤 |

### 过线标准（须同时满足适用项）

| 检查项 | 标准 | BE | FE/小程序 |
|--------|------|----|-----------|
| **编译构建** | lint/type 无 error + build 成功 | `mvn -q -DskipTests package`（或约定） | `build` / `build:weapp` 成功 |
| **能启能调** | 见下方「能启能调分档」（**不削弱**原可运行要求） | 见下 | 见下 |
| **主路径冒烟** | 本 T（或本模块）主路径不炸 | 该 T 的核心写/读接口：200/业务成功，不 500 | 该页能打开、核心按钮能走完（可人工点，须记录结果） |

#### 能启能调分档（堵「单测绿但起不来」· 不改流程）

| 场景 | 能启能调最低标准 |
|------|-------------|
| **该端（BE 或 FE）本阶段第 1 个 T 勾③前** | **必须真实启动**：BE 进程+health/探针 UP；FE `dev` 能起或开发者工具可开 |
| **同端后续单 T** | BE：health UP **或** 本 T API 用 curl/MockMvc/集成测打通 happy path（非 500）；FE：能启或关键页不白屏 |
| **F-xxx / MVP 切片确认前** | **必须真实启动**（同第 1 个 T） |
| **给用户看 / MVP 演示** | **必须真实启动**；**禁止**仅用 MockMvc 代替 |

> 同端第 1 个 T 强制真启，避免「一路 MockMvc 绿、切片才发现起不来」。中间 T 可用 MockMvc 保节奏；切片/演示仍须真启。


**与 ③ / 测试入场门禁的关系**：

| | 可运行闸门 | 步骤 ③（AC） | 阶段 5 测试入场门禁 |
|--|--------|--------------|----------------|
| 目的 | **能编、能启、主路径不炸** | Given/When/Then **对不对** | 进 tests 前**全量**再验一遍 |
| 深度 | 编译+探针/接口+轻量冒烟 | 1 AC ↔ 1 自动化测试 | 编译+启动+功能冒烟（全端） |
| 失败 | **禁止勾③ / 禁止说可给用户看** | 回修代码 | 禁止 AC 验收归档 |

### 证据（勾③前必须有 · A 档 runnable 会扫）

写入 dev **九**（权威；todo 可镜像）：

- 编译构建 / 能启能调 / 主路径冒烟 各一行：命令 + 结果（exit 0 / health UP / 冒烟通过）
- 若跑了 FE Playwright：附 `atlas/logs/fe-smoke-report.json` 的 summary（pass/fail/skip）
- **有强制原型**：勾③前跑 [fe-pixel-compare](fe-pixel-compare.md)；九写 `atlas/tests/fe-pixel/report.json` PASS
- **禁止**只写「测过了」无命令无结果；**禁止**空表「③ 验收后填写」就勾③
- 标「开发实现 ✅」前须 `--gate dev-complete`（含 runnable；有原型含 pixel）exit 0；进阶段 5 前须 `--gate test-entry` exit 0  
  → [validate-atlas-gate](validate-atlas-gate.md)

### ❌ 不过

- 只跑 `test/ac` 单测、从不 `package` / `build`  
- 编译红仍标 T ✅  
- 服务起不来 / 登录 500 / 首页白屏仍说「MVP 好了给用户看」  
- 把可运行闸门 推到「等阶段 5 再说」
