# 并行编排（可选）

> **默认不启用。** 须用户**原话**含「并行 / 多 subagent / 同时开 FE+BE / 同时开发 FE+BE」。  
> **「全部做 / 全部开发 / 直接全开发 / yes_all」≠ 并行许可，≠ 可派 Subagent 批量写码。**  
> **一句话「并行」不够** → 须过下方**并行启动卡**后才能启批量构思 / 写码执行批。

细则 → [04-development.md](04-development.md) · [dev-quickstart.md](../templates/dev-quickstart.md)

---

## 并行启动卡（强制）

用户说了并行类原话后，**启 Task / 批量勾①之前**必须 AskQuestion：

```yaml
title: "并行开发确认"
questions:
  - id: parallel_confirm
    prompt: |
      将启用并行：批量构思 先齐最多 3 个 T 的①，再 Subagent 各做 1 个 T 的②→可运行闸门→③。
      确认开闸？
    options:
      - id: yes_parallel
        label: "确认并行（本批最多 3 个 T）"
      - id: no_serial
        label: "还是串行逐 T"
```

选 `no_serial` / 未发本卡 → **禁止**并行。选 `yes_parallel` → 本批最多 **3** 个 T；下一批须再发并行启动卡。

## 硬否决（先读）

| 用户说了… | 可否启 Task/Subagent 写业务源码？ |
|-----------|----------------------------------|
| 全部做 / 全部开发 / 直接全开发 / 一次性做完 | **否** → 主 Agent **串行**逐 T |
| 并行 / 多 subagent（显式） | **仅当**并行启动卡已确认 **且** 每个切片已有合规 `atlas/dev/T-xxx.md` 且构思闸门 已过、① 已勾 |
| 未说并行 / 未过并行启动卡 | **否** |

**禁止**：

- 用 1～2 个 Subagent「实现全部 BE / 全部 FE / T-002～T-018」
- Subagent prompt 含多个 T 或「其余任务一并完成」
- 无 `atlas/dev/T-xxx-*.md` 就让 Subagent Write `backend/` / `miniapp/` / `src/` 等业务源码
- 主 Agent 自己不写 dev，把阶段 4 整包外包
- 单次批量构思 批量勾①超过 **3** 个 T
- Subagent 改 `atlas/todo.md` / 创建 `atlas/dev/`（发现即违规，主 Agent 须回滚勾选）

---

## 不豁免

- 每个切片仍须 **① → 构思闸门 → 勾① → 写码闸门 → ② → 可运行闸门 → ③**
- 催进度不可少结束闸门、不可跳过构思
- 禁止整包「实现全部 MVP」跳过 ①

## 阶段 4 并行序（仅显式并行 + 开闸后）

```
并行启动卡 → 停 → 用户确认
批量构思：主 Agent 写【本批≤3个】T 的 dev → 构思闸门 → 勾 ①
         ※ 本批①都勾完之前，禁止任何 ②
         ※ 若本批某 T 未过构思闸门：修复该 T → 重过闸门 → 其他 T 等待
         ※ 经 2 次修复仍不过：移出本批串行处理，其余 T 可先进写码批
写码执行批：每个 Subagent 只领【1 个】T 的 ②→可运行闸门→③
         → 主 Agent【回收验收】后才勾 todo（见下）
全部父任务 ✅ → 阶段闸门 → 停
```

**禁止**跳过批量构思 直接 ②。  
**禁止**跳过可运行闸门 勾③。  
**禁止**一个 Subagent 领取「T-002～T-005」或「全部 API」。  
**禁止**Subagent 创建/改写 `atlas/dev/T-xxx.md`（① 只属主 Agent）。

**中途切换**：批量构思完成后可从并行切串行（直接逐 T ②→③ 即可）；串行中切并行须重新发并行启动卡。

### 与串行闸门的关系（防冲突）

| 规则 | 串行默认 | 显式并行例外 |
|------|----------|--------------|
| **串行约束** | 禁止多份「①勾完②未完」；禁止批量①再写码 | 开闸后批量构思 允许先齐**≤3** 个 T 的①；写码执行批 再并行② |
| 「全部开发」 | 串行逐 T | **不**触发本例外 |

### 主 Agent 回收验收（Subagent 回报后 · 强制）

Subagent 结束后，主 Agent **在勾 ②/③ / 父任务 ✅ 之前**必须亲自核对：

| 检查项 | 检查 | 不通过 |
|--------|------|--------|
| **源码范围核对** | 业务源码 diff 落在该 T「## 做法」范围内 | 超范围 / 改了其他 T |
| **禁止改 atlas** | **未**改 `atlas/todo.md`、**未**新建/改写 `atlas/dev/` | 有 → 回滚 atlas 变更 |
| **可运行证据** | 可运行闸门证据可复现（命令+结果）；主 Agent 可抽查重跑 | 无证据 / 仅口头「过了」 |
| **主 Agent 勾选** | 再由**主 Agent**勾 TodoWrite + atlas/todo | Subagent 已勾的一律作废重勾 |

**禁止**「Subagent 说做完了」就代勾。

### Subagent 异常时的回收

Subagent 崩溃/超时/返回不完整结果时：

1. 检查 git status / git diff 确认 Subagent 已写入的文件
2. 半成品处理：回滚该 T 源码到 Subagent 开始前状态（git checkout 对应文件）
3. 该 T 改由主 Agent 串行重做（从②开始，①已由主 Agent 完成）
4. 同批次已通过验收的 T 保留，不回滚
5. 在 todo.md 标注 `T-xxx: Subagent 崩溃，已回滚改串行`

### 批次部分失败处理

本批 3 个 T 中部分通过、部分失败时：

1. **已通过验收的 T**：③可勾，源码保留
2. **失败的 T**：源码保留（不回滚，部分代码可能可用），③不勾，todo 标注 `T-xxx: 可运行闸门未过·待修`
3. **下一批**：不需要等失败 T 修好；失败 T 须在本批所有 T 回收后由主 Agent 串行修复
4. 修复失败 T 时：从②继续（①已由主 Agent 完成），修完过可运行闸门 → 勾③

## 启用条件（须全部满足）

1. 用户显式说「并行 / subagent / 同时开发 FE+BE / 同时开 FE+BE」  
2. **并行启动卡**已选确认  
3. solution 已确认  
4. 路径无冲突（见 [active-edits](../templates/active-edits.md)；写码批只锁**源码**，不锁①）  
5. **每个**将并行的 T 已有合规 `atlas/dev/T-xxx-*.md` 且 ① 已由**主 Agent**勾  
6. 本批 ≤3 个 T  

**「全部做完」≠ 并行许可。**  
**与 active-edits**：批量构思① = 主 Agent；Subagent 只在写码批登记业务文件。详见 active-edits「并行切片」。

---

## 角色

| 角色 | 做 | 禁止 |
|------|-----|------|
| 主 Agent | 并行启动卡、拆片（1 片=1 T）、**全部①**、构思闸门/写码闸门、回收验收（源码范围/禁止改文档/可运行证据/主Agent勾选）、全部 AskQuestion、todo 勾选 | 未开闸启 subagent；把「全部开发」当成并行；跳过结束闸门；把①外包；代勾不验收 |
| Subagent | **仅 ②→可运行闸门→③**（1 个 T）；prompt 含 **已存在** 的 `atlas/dev/T-xxx-*.md` 路径 | 写① / 创建 dev；跳过可运行闸门；多 T；自行标 todo ✅；AskQuestion；改 atlas/ |

---

## 启 Task 前主 Agent 自检

```
1. 用户是否显式要求并行/subagent？（「全部开发」不算）
2. 并行启动卡是否已确认 yes_parallel？
3. 本批是否 ≤3 个 T？本 Task 是否只含 1 个 T-xxx？
4. atlas/dev/T-xxx-*.md 是否已存在且构思闸门 已过、① 已勾？
任一项否 → 禁止 Task，改由主 Agent 串行
```

## subagent prompt 必含

```
任务：T-xxx（仅此一个 T）· dev 文件：atlas/dev/T-xxx-….md（必须先 Read）
只做：按「## 做法」写码 → 可运行闸门 → AC；禁止改 atlas/、写其他 T。
```

回报须含：改动文件列表 + 可运行闸门命令与结果（供主 Agent 验收）。
