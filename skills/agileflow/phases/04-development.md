# 阶段 4：开发实现

> **只写代码**（业务 + 单元/集成测试代码）。验收报告属于阶段 5，**本阶段禁止跑验收测试**。

## 模式

按 [00-intent-routing.md §模式](00-intent-routing.md) 判定的快速/严谨模式执行。

---

## 快速模式

1. 按 003 创建文件结构
2. 从 todo 取首个未完成任务 → 标进行中
3. 实现业务逻辑（含可观测性日志）
4. 本地编译验证
5. 标已完成，更新 todo
6. 重复至全部完成 → **AskQuestion 阶段闸门**（见 [askquestion-gate.md](../templates/askquestion-gate.md)）→ 停止

**规则**：符合编码规范；禁止 >100 行大函数；每任务立即更新 todo；禁止验收测试。

---

## 严谨模式（TDD）

1. 取未完成任务 → 进行中
2. RED：写失败测试 → 确认失败
3. GREEN：最小代码通过 + 关键日志
4. REFACTOR：保持通过
5. 自查日志覆盖 REQ 核心 Then
6. 标完成，更新 todo → 重复 → **阶段闸门** → 停止

**规则**：无测试不写代码；变更模块覆盖率见阶段 5 L4（≥80%）；禁止验收测试。

---

## 完成交还

1. todo「开发实现」→ ✅
2. 发 **AskQuestion 阶段闸门** → 停止，等待进入测试验收

**测试代码标注**：`REQ-XXX` + `场景N`；对照 003 §测试环境；mock 注释 `# L3: mock …`

---

## 可观测性

以 `003 §可观测性` + [observability-logging.md](../templates/observability-logging.md) 为准：

- event 名、字段、级别以 REQ→event 表为唯一依据
- 日志框架、目录、配置文件**随项目技术栈**，003 中已选型
- dev/prod 必须文件落盘；禁止仅 stdout
- L3 mock 按 003 §测试环境；禁止静默连生产 API
- 日志抽独立方法；关键分支注释业务意图
