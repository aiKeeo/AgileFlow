# Agileflow 交互示例

> 演示**节奏**（谁停、落什么）；业务名可换。  
> **默认常见路径**：`fast+ai`（委托交付）。user 分步停见示例 **C**。  
> 排障 → [TROUBLESHOOTING](../TROUBLESHOOTING.md)

---

## A. 新手第一次用

**用户**：我想做个小程序待办，不太懂流程，怎么开始？

**AI**：AskQuestion **流程启动卡**（快速/严谨 × 我来/AI自主）→ **停**（不写 REQ）  
**用户**：选「快速 + AI自主」  
**AI**：写 `agileflow.env` → 落盘 REQ + todo → 审阅摘要（fast+ai 可免卡）→ **停**  
**用户**：继续 → AI 下条进建模判定或 sol…

---

## B. 默认路径：快速 + AI自主（委托交付）

**用户**：做账号密码登录，你定，别问我。

**AI**：
```
📍 Agileflow | 模式：fast | 决策：ai
```
直接写 env → REQ（含 AC）→ 摘要确认 → **停**  
**用户**：继续 → sol（F 边界←REQ、AC→主 T、todo）→ **停**  
**用户**：继续 → TodoWrite → 按 T 做①②③…

> 少问 ≠ 跳阶段；①与可运行闸门仍强制。

---

## C. user 严谨（对照：更多停点）

**用户**：`req:` + 描述（契约已是严谨+user）  
**AI**：缺口卡（若有）→ **停** → 写 REQ → 确认卡 → **停** → 阶段闸门 → **停** → 用户「是，继续」→ 下条进下一阶段。

---

## D. 探索（不知道做什么）

**用户**：项目好慢，想优化，不知道瓶颈在哪。  
（或 `explore: 看看性能`）

**AI**：
```
📍 Agileflow | 探索模式
```
读代码 → 发现摘要 + 2～4 个方向 → AskQuestion → **停**（**不写 REQ/源码**）  
**用户**：选「先治接口 N+1」  
**AI**：再进 `req:` / 豁免，按正式流程。

---

## E. 中途改需求（L2）

**用户**：登录成功不要跳首页，改跳上次浏览页，AC 要改。

**AI**：`纠偏：L2 → REQ-001` → 改 REQ AC（按 change-management 留痕）→ 影响分析卡 → **停**  
**用户**：确认只动 FE  
**AI**：相关 T 重做③（必要时回①）→ 再验收。

---

## F. 并行 FE+BE

**用户**：登录前后端并行，用 subagent。

**AI**：确认 T-001/T-002 已有合规① → **并行启动卡** → **停**  
**用户**：确认  
**AI**：Task 各做一 T 的②→可运行→③；主 Agent 核对证据再勾。

> 无「并行」原话禁止 Subagent 写业务码。

---

## G. 豁免（修一行）

**用户**：首页标题拼错一字，不走流程。

**AI**：`📍 豁免：微型` → 直接改源码 → 不做 REQ/dev。  
多模块/API/DB **禁止**冒充豁免。

---

## H. humanTodo 阻塞 → 解除

**AI**：（sol）写入 humanTodo「提供支付商户号」→ 可标阻塞  
**用户**：已写入 `.env`  
**AI**：勾掉 humanTodo → 继续被阻步骤。

---

## I. 阶段 4 单 T：①②③

**用户**：做 T-001 登录 API。

**AI**：
1. TodoWrite：①构思 / ②写码 / ③AC  
2. 写 `atlas/dev/T-001-*-BE.md` → 勾①  
3. 按步骤写码 + UT（1 AC↔1）→ 勾②  
4. 真跑 build/curl → `## 结果` + AC 映射表 → 勾③  

禁止：无文件勾①；未跑命令勾③。

---

## J. 验收报告

**用户**：`tests:`  

**AI**：入场门禁（同会话可复用③）→ 写 `atlas/tests/REQ-*-验收报告.md` → 更新 `tests/README.md` → PASS / BLOCKED-HUMAN。

---

## K. 已有：方案 / init / 续跑（短）

| 场景 | 节奏 |
|------|------|
| **sol**（user） | 技术栈卡→停 → architecture+todo → 确认/闸门→停 |
| **sol**（fast+ai） | 自选栈落盘 → 摘要/免卡→停 |
| **init:** | 写法锚点卡→停 → 扫描落盘 init → 确认→停 |
| **继续 agileflow** | 读 todo checkpoint → 重建 TodoWrite → 接着做 |

---

## 违规反例

- ❌ 探索类问题直接写 REQ/源码  
- ❌ 首启未问就写 env/REQ  
- ❌ 无①写码；事后补写勾①  
- ❌ 勾满①②③却无 `dev/` 或无可运行证据  
- ❌ 「用户催了」跳过 A 档
