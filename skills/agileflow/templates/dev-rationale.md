# 构思落盘文档（atlas/dev/）

> **格式模板**：颗粒度以 [dev-exemplar-BE.md](../examples/dev-exemplar-BE.md) / [dev-exemplar-FE.md](../examples/dev-exemplar-FE.md) 为准；落盘时换成本任务内容。
> 阶段 4 **步骤 ①**。  
> **dev = 构思落盘，不是事后补文档。**  
> **没落盘 → 禁止写业务源码。**

## dev 规则：构思先行

**开发的第一件事是构思，第二件事才是写码。** ① 须按下方**九段模板**输出到 `atlas/dev/`（**严谨**一～九全写；**快速**可省略二/四/六/七整段，见 D3）。

| 规则 | 说明 |
|------|------|
| **D1** | dev 文档 = **构思落盘**；聊天里的思路不算，必须写文件 |
| **D2** | 一任务一构思：`T-{id}-{简述}-{BE\|FE}.md` |
| **D3** | 严谨一～九标题；快速一/五/八/九（FE+三）；二四六七快速可整段省略 |
| **D4** | 五须可执行（做法含 `` `Class.method` `` 或 pages/services）；禁空壳/「见 REQ」独撑 |
| **D8** | FE+UI：字面量 `### 3.1` `### 3.2` `### 3.3`；**3.1 须含 ASCII 布局线条图**（对齐 UID）；3.2 英文字段 |
| **D11** | 有 UI 的 FE / 有 API 的 BE：须有 `### 复用盘点（FE）` 或 `### 复用盘点（BE）`；路径来自本端锚点**资产索引**或「无」+新建理由（见 [code-conventions](code-conventions.md)） |
| **D5** | A0 Read exemplar→写1个T→A→①→B→②→**闸门C**→③；**默认禁止批量①**（显式并行批次 B 例外见 parallel） |
| **D6** | 补盘/摘要/空壳 = 无效；不勾① |
| **D7** | ①未勾禁止写码 |
| **D9** | 禁合并多T；**连续做≠连进模式≠压缩模板**（见 flow-modes） |
| **D10** | 闸门A：段标题+五可执行；T 头≥3或BE+FE强制严谨 |

architecture / contracts 讲**是什么**；**dev 讲这任务怎么构思、怎么写、怎么验**——不可互相替代。

细则：[04-development.md](../phases/04-development.md)

## 九段 → ②③ 怎么用

| 段 | 章节 | 构思时必答 | ②③ 怎么用 |
|----|------|------------|-----------|
| 理解 | **一** | 核心目标、必须满足、**明确不做** | ② 不越界 |
| 数据 | **二** | 表/实体/字段/约束 | ② 建表、Entity |
| 契约 | **三** | BE：接口+**复用盘点（BE）**；**FE+UI**：线框+**复用盘点（FE）**+映射表+调用清单 | ② 实现 API / 按表绑 UI；**先复用再新建** |
| 状态 | **四** | 状态流转（无则写「无」） | ② 状态校验 |
| 流程 | **五** | **目的+需要什么+怎么做(5.x)**；做法含复用路径 | ② **按 5.x 做法**写码 |
| 边界 | **六** | 并发、幂等、异常场景 | ② 异常处理 |
| 依赖 | **七** | 资产索引行 + §3.x 参考页 + 假设 | ② 按栈实现 |
| 验收 | **八** | REQ AC + test/ac 规划名 | ③ 只跑此表 |
| 结果 | **九** | 表头 ① 必建 | ③ 回填 L3 |

**快速模式**：**一、五、八、九不可省**；**FE+UI 时三也不可省**（须 `### 3.1/3.2/3.3`，3.1 含线条图）；二/四/六/七可整段省略（省略则不写标题）。权威 → [flow-modes.md](flow-modes.md)。  
**禁止** FE+UI 任务把三写成「无」。

## 铁律

| 铁律 | 说明 |
|------|------|
| **构思先于写码** | 无 dev → 禁止 Write 业务源码 |
| **一任务一构思** | 1 个 T-xxx = 1 个 dev 文件 |
| **段数齐全** | **严谨**：一～九标题+正文；**快速**：一/五/八/九（FE+三）；二四六七可省略 |
| **五、核心流程逐步** | 禁止「实现某某模块」一句带过 |
| **顺序** | ① 构思落盘 → 闸门A → 勾① → 闸门B → ② 按 **五** 写码 → **闸门C** → ③ 按 **八** 验收 |

## 构思反模式

| ❌ 反模式 | ✅ 正确 |
|----------|---------|
| 口头想想直接写码 | Read exemplar → 完整模板 → 闸门 A（**含 A7 grep**）→ 勾① |
| 摘要 / 空壳九标题 | 五可执行（含类·方法痕迹） |
| `## 一、目标` + `## 五、可执行方案` 列表 | 模板「一、需求理解」+「五、核心流程」+ `### 目的`/`#### 5.` |
| 有文件未 grep 就勾① | A7 命中后再勾 |
| 不查资产索引就新建平行组件/Util | Read 本端锚点资产索引 → 复用盘点 → 再 5.x |
| 「尽量复用」无表 | `### 复用盘点（FE|BE）` 短表 |
| 批量写完所有①再写码（无并行许可） | 串行：同一时刻只推进 1 个 T 的①→②；显式并行见批次 B |
| T≥3 自称快速 | 强制严谨 |
| 「见 REQ」代替五 | 写法步骤到 `Class.method` |

## 五、核心流程颗粒度（W3 检查）

| 要求 | 门槛 |
|------|------|
| **五** 结构 | **目的** + **需要什么**（表）+ **怎么做**（≥1 个 `5.x` 小节） |
| 每个 **5.x** | **要达成什么** + **需要什么** + **做法**（BE→类/方法；FE→页面/组件/方法） |
| 绑定 AC | 5.x 标题或验收表链 REQ 的 AC ID |
| 禁止 | 「实现 XX 模块」、无具体落点的空步骤 |
| 对照 | 与对应端 exemplar 同级或更细；**FE+UI** 时 **三、3.2** 每行可在 `API-xxx` 搜到字段名 |

省略条件（极少）→ [04-development.md](../phases/04-development.md)。
