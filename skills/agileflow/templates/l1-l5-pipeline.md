# L1–L5 测试流水线

> Agileflow 各阶段引用本文件为权威定义。

## 流水线顺序

```
0a 人类资源闸门 → 0b 资源文件 → 0c 技术依赖 → L1 → L2 → L3 → L4 → L5 → 逐 REQ 验收
```

步骤 0 详见 [human-todo.md](human-todo.md)。

## L1–L5 简表

| 层 | 通过标准 | 快速模式 | 严谨模式 |
|----|----------|----------|----------|
| L1 | lint/type 无 error | ✅ | ✅ |
| L2 | build 成功或 N/A | ✅ | ✅ |
| L3 | 含 REQ 用例，全过 | ✅ | ✅ |
| L4 | 覆盖率见下 | 仅报告 | ≥80% 变更模块 |
| L5 | 冒烟/E2E（真连 003/humanTodo 外部资源） | humanTodo 未齐则不得标 pass | 核心不得无故 skip |

**L4 严谨**：变更模块 ≥80%；支付/权限/写入分支不得 0%。

**快速模式 L5**：可 skip 并记原因；humanTodo 未齐时不得标 pass。

## 豁免路径（微型改动 / Hotfix）

| 豁免类型 | 测试要求 |
|----------|----------|
| 微型改动 | 改代码后跑 **L1 + L3**（豁免分支执行，非阶段 4） |
| Hotfix | **L1 + L3** 必做；核心路径 **L1–L3 + L5 冒烟** |

> 阶段 4 禁止跑验收测试；豁免 L1+L3 在 [00-intent-routing.md §①](../phases/00-intent-routing.md) 或阶段 5 前执行。

## 失败重试

AI 亲自跑终端；从失败层重跑，最多 3 轮；仍失败 → FAIL，回阶段 4。

## 取证要求

按 `003 §可观测性` 的 event 名在**日志文件**检索；附 traceId + 片段 + **文件路径**。
