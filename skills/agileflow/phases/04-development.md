## dev 规则：构思先行（最高优先级）

> **开发的第一件事不是写码，是构思落盘。**  
> `specs/dev/T-xxx.md` = 本任务的构思文档；②③ 都是执行这份构思。

| 规则 | 说明 |
|------|------|
| **R1 构思先于写码** | 没 dev 文档 → **禁止** Write 业务源码（W2） |
| **R2 构思必须落盘** | 构思只放脑子里或聊天里 → **无效**；必须写进 `specs/dev/` |
| **R3 一任务一构思** | 1 个 T-xxx = 1 个 dev 文件；禁止多任务糅合 |
| **R4 七段齐全** | 一～七 + **八**（九表头）；格式见 dev-rationale |
| **R5 五、核心流程是构思核心** | 逐步编号主逻辑；禁止「实现某某模块」 |
| **R6 ② 只执行 五** | 写码 = 把 **五、核心流程** 变成代码；禁止脱离构思即兴发挥 |
| **R7 ③ 只验 八** | 验收 = 对照 REQ AC；禁止在 dev/test 里自造 AC |
| **R8 顺序不可跳** | ①→②→③；催进度 / 委派 / 并行 **不豁免** |

**构思的目的**：接手人读 dev 就能懂——**五** 知怎么写、**八** 知怎么验、**一** 知边界、**七** 知缺什么 humanTodo。

---

## 开发三步序

```
① 构思落盘          ② 按构思开发           ③ 对照 REQ 验收 AC
────────────────    ────────────────       ─────────────────────
七段写全、落盘        严格按 五 写码          跑 八 规划的 test/ac
specs/dev/           注释对齐构思             全绿 → 九 回填 → todo ✅
```

| 违规 | 后果 |
|------|------|
| 没 ① 就 ② | 构思未完成；Write 拦截 W2 |
| 没 ② 就 ③ | 无被测代码 |
| ③ 自造标准 | AC 只在 REQ |
| 构思写一半就写码 | W3/W4 拦截 |

---

## ① 构思落盘：七段齐全

一任务一文件：`specs/dev/T-{id}-{简述}-{BE|FE}.md`

**必须按 [dev-rationale 七段构思模板](../templates/dev-rationale.md#七段构思模板-①-必须按此格式输出) 输出**：

| 段 | 章节 | 构思必答 |
|----|------|----------|
| 理解 | **一** | 核心目标、必须满足、**明确不做** |
| 数据 | **二** | 表/实体/字段/约束（FE 可写组件状态） |
| 契约 | **三** | 接口、入参、出参、异常码 |
| 状态 | **四** | 状态机（无则写「无」） |
| 流程 | **五** | **逐步主逻辑**（② 写码蓝图） |
| 边界 | **六** | 并发、幂等、异常场景 |
| 依赖 | **七** | 框架、中间件、假设、humanTodo |
| 验收 | **八** | REQ AC + test/ac 规划名 |
| 结果 | **九** | 表头 ① 必建，③ 后填 L3 |

**禁止**：只写一+五；缺六/八；多 T 糅合一文件；写码后再补 dev。

快速模式：**一、五、八不可省**；二/三/四/六/七 按任务裁剪。

构思自检清单 → [dev-rationale 构思检查清单](../templates/dev-rationale.md#构思检查清单①-必过)

## Write 拦截（构思未完成 → 禁止写码）

| # | 检查（构思维度） | 否 → |
|---|------------------|------|
| W0 | `specs/todo.md` 该 T-xxx **① 已勾**；`TodoWrite` ① 已 ✅ | **先完成 ① 构思并勾 todo** |
| W1 | 单个 T-xxx | 读 todo |
| W2 | **构思已落盘**：`T-{id}-*.md` 存在 | **先完成 ① 构思** |
| W3 | 七段齐全（一～七）+ **八** + **九表头** | 补全构思 |
| W4 | **五、核心流程**逐步展开；**六** 有异常表 | 重写五/六 |
| W5 | G0–G8 构思闸门通过 | 停止 |
| W6 | Write 内容在 **五** 当前步骤范围内 | 先更新 dev **五** 再写 |
| W7 | **七** humanTodo 已沉淀 | 先追加 humanTodo |

## ② 按构思开发

- **只做一件事**：把 **五、核心流程** 变成代码
- 严格按 **五** 步序，一步一动作
- 代码注释对齐构思（数据模型、状态约束写进代码）
- **禁止**脱离 **五** 即兴扩 scope（那是新构思，须回 ①）

---

## ③ 对照 REQ 验收 AC

- **只做一件事**：验证 **八** 里的 REQ AC 是否满足
- 只测 **八** 列出的 AC；**禁止自造 AC**
- 在 `test/ac/` 写/跑 **八** 规划的用例 → **全绿**
- **九** 回填 L3 + 终端证据
- **任务 ✅** = **九** 可自动化 AC 全 🟢

细则 → [ac-guide.md](../templates/ac-guide.md)

---

## 构思反模式（常见违规）

| 反模式 | 正确做法 |
|--------|----------|
| 直接写码，事后补 dev | 先 ① 构思落盘，再 ② |
| dev 只写「实现登录模块」 | **五** 逐步编号主逻辑 |
| 多个 T-xxx 写一个 dev | 一任务一构思文档 |
| **八** 自造 AC | 只引用 REQ AC |
| 聊天里说说思路不落盘 | 写进 `specs/dev/T-xxx.md` |
| ② 写码时临时改方案不更新 **五** | 改构思 → 先更新 dev，再写码 |
| ③ 未全绿就标 todo ✅ | **九** 须全 🟢 |

## 默认路径（单 Agent 串行）

1. `specs/todo.md` 取首个 **① 未勾** 的 T-xxx → 写入「进行中」（含当前步）  
2. **`TodoWrite` 同步该任务三步子项**（①构思落盘 / ②按五写码 / ③对照REQ验收AC）  
3. **只做 ①**：按七段模板构思落盘 → G0–G8 → **勾 todo ① + TodoWrite ① ✅**  
4. **只做 ②**：按 **五** 写码 → **勾 todo ② + TodoWrite ② ✅**  
5. **只做 ③**：对照 REQ 验收 → **九** → **勾 todo ③ + TodoWrite ③ ✅** → 父任务 ✅  
6. 下一任务 → **新 dev 文档**（禁止预写 T+n）

> **硬约束**：todo ① 未勾 / TodoWrite ① 未完成 → **禁止 Write 业务源码**（W2 叠加）。

### 阶段收尾

全部 todo ✅ → **AskQuestion 阶段闸门** → 停止

## G0–G8（构思闸门：① 完成后、② 开始前）

| # | 构思检查 |
|---|----------|
| G0 | dev 已建（一任务一构思文档） |
| G1 | **一** 需求理解三项齐全 |
| G2 | **二** 数据模型/组件状态已列 |
| G3 | **三** 接口契约已列（有 API 时） |
| G4 | **五** 核心流程可执行；**六** 异常边界已列 |
| G5 | todo 进行中 |
| G6 | 全栈当前端定稿 |
| G7 | **五** 逐步达标（见 [dev-rationale 五、核心流程颗粒度](../templates/dev-rationale.md#五核心流程颗粒度-c5-检查)） |
| G8 | **八** AC 对照 + **九** 表头已建 |

**G0–G8 任一不通过 = 构思未完成 = 禁止 ② 写码。**

## humanTodo

构思 **七** 或 ② 中发现人类依赖 → **当次追加** humanTodo。阶段 4 开始前做阻塞检查。

## 委派

未显式并行/subagent → 禁止 Task。subagent 须附 **单 T-xxx + ① 构思文档路径**；仍走 ①→②→③。
