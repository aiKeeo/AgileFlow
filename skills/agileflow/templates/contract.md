# 流程契约（模式 × 决策权）

> **唯一 SSOT**：首启、决策权、停点、话术、AskQuestion 卡。他处只链本文，禁止再抄表。  
> env 字段 → [agileflow.env](agileflow.env) · 派活 → [orchestrator](orchestrator.md) · 冲突 → [SKILL 裁决表](../SKILL.md#裁决表冲突时以此为准)

---

## 0. 四种组合（人话）

| 组合 | 你感受到的 |
|------|------------|
| **连做 + 我来** | 少停；关键点仍问你；确认+闸门可合并 |
| **连做 + AI 全权** | 很少问；闸门绿后可连做下一阶段；阶段 4 推并发 |
| **严谨 + 我来** | 分步确认；一阶段一停 |
| **严谨 + AI 全权** | 不问选型，但每阶段结束要你审阅（可说「不想看了」改连做+AI） |

> **命名**：用户向称 **连做模式**（`AF_FLOW=fast`）；旧称「快速模式」同义。**连做 ≠ 少写文档**，只少停点 + 可并发。

**只有 `AF_DECIDE=ai` 才是全权。** `user` 须问缺口/确认。决策权不锁死：中途可让 AI 接管或收回。

---

## 1. 两维 → env

| 维 | 值 | 控制 | 不控制 |
|----|-----|------|--------|
| `AF_FLOW` | `fast` / `strict` / `pending` | 停点、连做、是否推并发 | sol/dev 厚度、①、闸门 |
| `AF_DECIDE` | `ai` / `user` / `pending` | 选型/澄清问不问人 | 是否落盘 atlas |

**铁律**（SSOT，他处只链）：**连做模式**（`AF_FLOW=fast`）≠ 跳阶段 ≠ 薄文档。加速 = 少停 + 并发，不是少写。

## 2. 首启（默认问人）

一开始 **空着**（`pending`）。从原话解析：

| 原话 | 动作 |
|------|------|
| **未**点明谁决策（只说做功能/带前缀） | 保持 pending → [启动卡](#71-流程启动卡)→**停**。**禁止**静默 `fast+ai` |
| 你定 / 别问我 / 直接做完 / 不用管 / AI自己干 | `AF_DECIDE=ai`（未说严谨 → 常配 `fast`）；可跳卡 |
| 严谨+你定 | `strict+ai`；可跳卡；阶段结束审阅→停 |
| 我来决策 / 逐步问我 | `AF_DECIDE=user`；模式未齐 → 启动卡（或只问模式） |
| 快速+你定 / 严谨+我来 等两边都齐 | 按原话写 env；不问 |
| 只说快速或只说严谨 | 仍问缺的决策维 |

答完启动卡（下条）：写 `AF_FLOW`/`AF_DECIDE`（禁再 pending）→ 同步 todo「决策委派」→ 按契约进阶段。启动卡本回复必须停。

### 契约重选

说「重选模式 / 重开启动卡 / 重新选决策权」或闸门选重选 → 先写 pending → 再发启动卡→停。禁止让用户手改 env。

## 3. 话术表（必须看上下文）

| 用户说 | 上下文 | 动作 |
|--------|--------|------|
| 不审了继续 / 跳过审阅 | **正在**审阅卡上 | 只 `skip_review`；**不**改 `AF_DECIDE` |
| 这阶段你定 / 不用问我了 | 任意 | 仅本阶段少问；**不**改全局 `AF_DECIDE`（全局仍可为 `user`） |
| 后面都你定 / 不想看了 / 剩下你来 / 直接做完 / 你接管 | 任意 | **立刻** `AF_DECIDE=ai`（想少停则 `AF_FLOW=fast`）→ 接管剩余；仍须闸门 |
| 这阶段我来 / 后面都我来 | 任意 | 切回 `user`（当阶段或全局） |
| 重选模式 / 重开启动卡 | 任意 | [契约重选](#2-首启默认问人) |
| 我自己看，暂停 | 审阅/闸门 | pause，不进下一阶段 |

**禁止**：口头答应接管却不改 env。

## 4. 停点总表

| | `user` | `strict+ai` | `fast+ai` |
|--|--------|-------------|-----------|
| 阶段内澄清/技术栈等 | 按缺口问 | **跳过**（自决+AI 决策记录） | **跳过** |
| 阶段结束 | 阶段闸门→停（快速可确认+闸门合并） | 审阅卡→停 | 闸门绿→一行摘要+自动确认→**可同回复连做** |
| 阶段 4 并行 | 候选≥2 → 并行卡 | 默认串行；显式才开 | 必扫描；≥2 自动并行 |
| F/MVP 里程碑卡 | F 齐且可运行过→**必须**问 | 默认不问 | 默认不问 |

AskQuestion 后 = **本回复结束**；用户答后 **下条必须落盘/进阶**。

---

## 5. 信息充分少问（user）

REQ 四项齐（用户/场景、平台、MVP 范围、功能清单可拆）→ 跳过**需求澄清整卡**，仍须确认/闸门。  
原话已覆盖的题禁止复问。`ai`：不问澄清/确认/技术栈/并行/init 锚点。

---

## 6. AI 自主时

跳过阶段内决策卡；须落盘 +「AI 决策记录」；仍不可跳：主链、atlas、todo①②③、构思①、可运行闸门、每阶段 validate exit 0。

`fast+ai`+闸门绿：禁发审阅卡；摘要；自动标已确认（阶段 1 同步各 REQ）；**允许同回复连做**。  
`strict+ai`：必须审阅卡→停（可[接管](#3-话术表必须看上下文)改 fast+ai）。

## 7. AskQuestion 卡册

### 7.1 流程启动卡

```yaml
title: "Agileflow 流程启动"
questions:
  - id: flow_mode
    prompt: "流程节奏？（只影响停点/并发；不减 sol/dev 质量）"
    options:
      - id: fast
        label: "连做（少停；AI全权时可自动进下一阶段；阶段4可并行，文档不减质）"
      - id: strict
        label: "严谨（一阶段一停；AI全权仍要审阅卡）"
  - id: decide_mode
    prompt: "关键选型谁定？（中途可说「后面都你定」或「这阶段我来」）"
    options:
      - id: user_decide
        label: "我来决策"
      - id: ai_decide
        label: "AI 全权决策"
```

### 7.2 阶段闸门（user）

```yaml
title: "开发流程确认"
questions:
  - id: continue_next_stage
    prompt: "{当前阶段}已完成。是否进入【{下一阶段}】？"
    options:
      - id: yes
        label: "是，继续"
      - id: no
        label: "否，暂停"
      - id: reselect_contract
        label: "重选流程契约"
```

快速+user 阶段 1/3：确认题与 continue 可合并为一张卡（仍本回复停）。

### 7.3 审阅闸门（strict+ai）

```yaml
title: "{阶段名} - 产出审阅"
questions:
  - id: review_action
    prompt: "已写入：{文件摘要}。AI决策：{要点}。阶段4须含可运行证据。"
    options:
      - id: confirm_continue
        label: "看过了，继续"
      - id: skip_review_continue
        label: "不审了，继续"
      - id: need_changes
        label: "我要调整"
      - id: reselect_contract
        label: "重选流程契约"
      - id: pause
        label: "暂停，先自己看"
```

### 7.4 F/MVP 阶段性确认（仅 user；ai 默认不发）

F 相关 T 全 ✅ 且可运行闸门已过 → 必须问：继续 / 先演示 / 有问题先修 / 暂停。可运行未过禁止发本卡。

### 7.5 并行启动卡（仅 user；fast+ai 不问）

候选 ≥2 可并发 T → 问是否本批并行。规则 → [04 §并行](../phases/04-development.md#并行阶段-4)。

### 7.6 其它卡（YAML 要点）

| 卡 | 何时 | 要点 |
|----|------|------|
| 需求缺口 | user 且四项不齐 | 只问缺口；充分则跳过 |
| 需求确认 | user 写完 REQ | 快速可与闸门合并 |
| 技术栈 | user 且栈 pending | 未落盘 architecture 前禁发方案确认 |
| 建模确认/跳过 | user | ai 自判落盘，禁确认卡 |
| 变更 | 改已确认 REQ | 见 [change](../phases/change-management.md) |

AskQuestion 不可用：编号选项降级 → 停 → 首行 `⚠️ AskQuestion 降级`。

---

## 8. 禁止

- 契约未确认静默写 `fast+ai`
- 对 `ai` 再发澄清/技术栈/方案确认/阶段闸门（strict 只留审阅）
- 只发卡不落盘；选「继续」后同回复不写下一阶段（`fast+ai` 连做除外）
- 可运行未过就发 F 里程碑卡或说「给用户看」
