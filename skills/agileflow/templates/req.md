# 需求文档输出模板

> **格式模板**：示例中的功能名/页面名仅为演示结构，落盘时替换为**当前项目**内容。  
> AC 规范（颗粒度≈单测、观测面）：[../phases/05-testing.md](../phases/05-testing.md)

## 不可删清单（快速模式同样适用）

以下节即使在快速模式下也**必须存在**，否则 `req-confirm` 闸门失败：

- `[REQ-XXX] 标题`（含编号方括号 + **实质需求名**；禁空名）
- 文件名 `REQ-XXX-名称.md`（须含名称后缀）
- `- 版本：` / `- 状态：` 元数据行（冒号后有值；状态枚举：草稿/已确认/已实现/变更中/已废弃）
- `## 范围提示` 节（**范围内/范围外**各自非空、非占位符）
- `## 验收标准（AC）` 节：完整 Markdown 表 = **表头 + `|---|` 分隔线 + 数据行**；列齐全（AC ID·场景·Given·When·Then·观测面·AC 测试方法·状态）；单元格非空

**硬规则**：空壳 / 占位符（……/待填/TODO）/ **省略表分隔线** / 列错位 = **闸门失败**。与 `AF_DECIDE=ai|user`、`AF_FLOW=fast|strict` **无关**——格式恒最严谨；ai 自主只决定「做什么」，不降格式。

**可压缩（非必须）**：业务价值大段、用户故事详细叙述、界面描述、非功能需求、版本历史。

## 验收标准（AC）

> **本表即 BDD**（Given/When/Then）。禁止再写独立「BDD 验收场景」节。  
> **颗粒度**：一条 AC ≈ 一条可独立红/绿的单测；BE 开发默认 `1 AC ↔ 1 UT`。  
> **观测面**：`API` | `UI` | `规则` | `人工`（只定从哪观察，不写测法/类名）。  
> 条数：常见 5–10；>12 先拆 REQ 或合并同质 AC。规则 → [../phases/05-testing.md](../phases/05-testing.md)。

| AC ID | 场景 | Given | When | Then（可断言） | 观测面 | AC 测试方法 | 状态 |
|-------|------|-------|------|----------------|--------|-------------|------|
| AC-001-01 | {主成功路径} | {前置} | {动作} | {可断言结果} | API | （③ 后填） | ⬜ |
| AC-001-02 | {失败/边界} | {前置} | {动作} | {可断言结果} | API | （③ 后填） | ⬜ |
| AC-001-03 | {规则/UI 观测} | {前置} | {动作} | {可断言结果} | 规则 | （③ 后填） | ⬜ |

## 界面描述（可选 — 有 UI 时填写）

| UID | 页面/流程 | 文档 | 样式状态 |
