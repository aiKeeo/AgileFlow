# Agileflow 交互示例

> 下列「用户」话术仅演示**节奏**；具体业务以用户当次原话为准，示例不写死领域。

## 示例 0：首启（必须先问模式+决策）

**用户**：描述要做的东西（未说模式、未说谁决策）

→ **回复 1**：AskQuestion **流程启动卡**（快速/严谨 + 我来/AI自主）→ **停止**（禁止写 REQ）  
→ **用户点选**  
→ **回复 2**：写 env（真实 `AF_FLOW`/`AF_DECIDE`）→ 按选定路径进阶段 1（`ai` 落盘+审阅 / `user` 需求卡）

**可跳过启动卡**：用户原话已同时点明，如「快速，后面都你定，做登录」→ 直接写 env 并落盘。

**旧项目重选**：用户说「重选模式」或审阅/阶段闸门选「重选流程契约」→ env 回 pending → 再发启动卡（不必手改文件）。

---

## 示例 1：需求（单阶段）

### 严谨模式（3 停 · 契约已确认 + user_decide）

**用户**：`req:` + 用户描述要做的功能（原话）

→ **回复 1**：AskQuestion 需求卡片 → **停止**  
→ **用户点选**  
→ **回复 2**：写 `REQ-xxx.md` → AskQuestion 确认 → **停止**  
→ **用户确认** → 标已确认 → **阶段闸门** → **停止**  
→ **用户选「是，继续」** → **下条回复**写 model/ 或 solution/

### 快速模式（2 停 · user_decide）

**用户**：`req:` + 已附完整需求描述

→ **回复 1**：写 `REQ-xxx.md` → **确认+闸门合并卡** → **停止**  
→ **用户确认+选继续** → **下条回复**写 model/ 或 solution/

## 示例 2：方案

**用户**：`sol:` + 用户要设计的范围

→ features + contracts → AskQuestion 技术栈 → **停**  
→ architecture + code-patterns + todo → 方案确认 → **阶段闸门** → **停**

（快速：技术栈 → 落盘 architecture → 确认+继续合并卡，见 flow-modes.md）

---

## 示例 3：开发（串行）

**用户**：阶段闸门已选「是，继续」→ 进入 T-001

→ **①** `atlas/dev/T-001-BE.md` → 构思闸门 → 勾 todo ① → 写码闸门 → ②
→ **②** 按 **## 做法** 写码 → 勾 todo ②
→ **③** test/ac 全绿 → T-001 ✅

---

## 示例 4：init（接手已有代码库）

**用户**：`init:`

→ **写法锚点模式卡**（A/B）→ **停**  
→ 用户选定 → 扫描源码 → 落盘 init/  
→ 自检全 ✅ → AskQuestion 确认 → **停止**

---

## 示例 5：AI 自主（审阅可跳过 · 契约已是 ai）

**用户**：描述要做的东西 + 「这阶段你定」（或启动卡已选 AI）

→ 写 REQ + `AI 决策记录` → **审阅闸门** → **停**  
→ 用户选「不审了，直接继续」→ **下条**写 solution/

---

## 违规反例

- ❌ 用户描述需求 → Agent **直接写业务源码**（须先落盘 REQ；AI自主→审阅闸门，user_decide→先澄清卡）  
- ❌ **首启未问模式/决策就写 env 或落盘 REQ**  
- ❌ 已确认 AI自主却只发卡不写 REQ  
- ❌ 没 ① dev 构思就写码  
- ❌ AI 自主但不写 atlas/  
- ❌ 用户选「不审继续」后只说不写
