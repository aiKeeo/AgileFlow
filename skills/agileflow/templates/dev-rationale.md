# 构思落盘（atlas/dev/）

> 颗粒度 → [exemplar-BE](../examples/dev-exemplar-BE.md) · [exemplar-FE](../examples/dev-exemplar-FE.md)  
> 档位 → [04](../phases/04-development.md) · 裁决 → [SKILL](../SKILL.md#裁决表冲突时以此为准)  
> **铁律**：只链不抄；每 `####` 步须有 `目的：`。

## 留什么

| 段 | 精简 | 标准·完整 | 写什么 |
|----|:----:|:--------:|--------|
| **## 前置** | — | ✅ | depends_on（为何）/ 运行条件 / 前提假设 |
| **## 必读** | — | ✅ | 表：用途 \| 链接 \| 本 T 用到什么（**禁粘贴** API/UID 正文） |
| **## 范围** | ✅ | ✅ | 目标 / 必须 / 不做 |
| **## 契约** | — | ✅ | **链** API/UID；FE：`### 布局`链 UID + `### 映射`；BE：复用表 |
| **## 做法** | ✅ | ✅ | `#### 名 — 目的：…` + 引用 + 做 + 完成标志 + 代码落点 |
| **## AC** | — | ✅ | AC→test（Then 一句话或链 REQ，**勿抄整段 Given/When**） |
| **## 结果** | ✅ | ✅ | ③ 填可运行证据 |

**不留**：一二三编号、空「状态机/异常/技术选型」段、抄 model/API/UID 全文、无目的的步骤列表。

## 骨架（标准）

```markdown
# [T-id] 名 — 构思 [BE|FE]
- 档位：[精简|标准|完整] · → [REQ]() · [API|UID]()

## 前置
- depends_on：T-xxx（为何必须先完成）
- 运行条件：…
- 前提假设：…

## 必读（只链，打开即用）
| 用途 | 链接 | 本 T 用到什么 |
|------|------|---------------|
| 验收 | [REQ-00x](…) | AC-00x-01/02 |
| 接口 | [API-00x](…) | 动作 add/list |
| 布局 | [UID-00x](…) | （FE） |
| 模型 | [model](…) | 本 T 碰的实体/表 |

## 范围
- **目标** / **必须** / **不做**

## 契约
→ [API-xxx](...)（勿重贴入参出参）
### 布局          # FE：→ [UID-xxx](...)；偏离才写「布局差量」
### 映射 / 复用 / 调用

## 做法
#### 步骤名 — 目的：一句话 `Class.method`
- 引用：API-003 §请求 / AC-002-01
- 做：…
- 完成标志：…

## AC
| AC | Then | test |

## 结果
| … | 编译+启/冒烟+PASS |
```

### 做法标题格式（闸门可 grep）

`#### {步骤名} — 目的：{业务/技术目的}` + 同行或下行代码落点 `` `Class.method` `` / `` `path/` ``

| 档 | 最少带 `目的：` 的 `####` |
|----|---------------------------|
| 精简 | ≥1 |
| 标准 | ≥2 |
| 完整 | ≥3 |

勾①：完整档另跑 `--gate dev-step1-literal --dev-file …`
