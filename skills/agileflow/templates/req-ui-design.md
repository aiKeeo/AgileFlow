# REQ 阶段 UI 设计描述模板

> **定位**：REQ 阶段描述**界面做什么、怎么交互**——供后续方案 `contracts/UI-xxx` 与 FE 开发生成 UI。  
> **不定义视觉样式**（配色、字体、圆角、动效细节等）——**样式由用户后续决定**（方案或开发前 AskQuestion / humanTodo）。  
> AC 规范：[ac-guide.md](ac-guide.md)

---

## 何时新建 UID 文档

| 条件 | 动作 |
|------|------|
| 有 Web / App / 小程序等**用户可见界面** | 评估是否需 UID |
| 页面/流程 ≥1 且交互非 trivial | **新建** `atlas/requirements/ui/UID-xxx-名称.md` |
| 纯 API / CLI / 后台批处理、无 UI | **跳过**，REQ 注明「无 UI」 |
| 用户说「样式我自己定 / 先不定样式」 | 仍写 UID（结构与交互）；**样式节只写「待定」** |

**一页面/一主流程 ≈ 一份 UID**；复杂模块可拆多份，REQ 头部链全部 UID。

|------|------|------|
| **UID-xxx** | UI **设计描述**（结构、交互） | 阶段 1 REQ |

## 2. 布局结构（信息架构，非视觉样式）

> **必须同时有：区域表 + ASCII 线条图（线框）。** 纯文字/纯表格 = 不合格。  
> 用区块描述信息架构，**不写**具体颜色、字号、间距数值。

### 2.1 区域表

| 区域 | 内容 | 说明 |
|------|------|------|
| 顶栏 | Logo、标题 | 可选 |
| 主区 | 登录表单 | 见「关键组件与交互」 |
| 底栏 | 协议链接 | 可选 |

### 2.2 布局线条图（强制 · ASCII 线框）

> **标题须可 grep**：`### 2.2` 或字面量 `布局线条图`。  
> 用 `┌─┐│└┘├┤` 画出上→下、左→右的区块与控件占位；**至少覆盖主路径可见控件**。  
> 禁止：只有「顶栏/主区/底栏」三个词；禁止用散文代替线框；禁止空代码块。

```
┌─────────────────────────┐
│         Logo            │
├─────────────────────────┤
│   [ 手机号输入框      ] │
│   [ 验证码输入框 [获取] ] │
│   [    登录按钮       ] │
├─────────────────────────┤
│   用户协议 | 隐私政策    │
└─────────────────────────┘
```

**复杂页**（多 Tab / 左右栏 / 列表+详情）：线框须体现分区关系（如 Tab 行、左列表右详情），可拆 2 张线框（默认态 / 关键态）。

## 3. 关键组件与交互

| 组件 | 类型 | 交互行为 | 校验/限制 |
|------|------|----------|-----------|
| 手机号 | 输入框 | 11 位数字 | 非空、格式 |
| 验证码 | 输入框 + 按钮 | 点击「获取」→ 60s 倒计时 | 需先填手机号 |
| 登录 | 主按钮 | 提交 → loading → 成功/失败反馈 | 表单合法才可点 |

## 阶段衔接

| 阶段 | 如何使用 UID |
|------|--------------|
| **1 REQ** | 新建/维护 UID；**§2 必须含区域表 + 布局线条图**；样式标「待定」 |
| **3 sol** | 读 UID → 生成 `contracts/UI-xxx`（**须转写/保留线条图** + 路由、组件树、API 绑定）；AskQuestion **样式/设计系统** |
| **4 dev FE** | Read `UID-xxx` + `UI-xxx`；dev **### 3.1** 须含布局线条图（可引用 UID 并按实现微调）+ 区域表；**3.2** 英文字段映射；② 按图与映射表绑 UI |

## UID 落盘自检（勾确认前 · 可机械检查）

| # | 检查 | 不过 → |
|---|------|--------|
| U1 | 存在 `## 2.` 布局结构 | 补节 |
| U2 | 有区域表（`\| 区域 \|`） | 补表 |
| U3 | 有 **`### 2.2`** 或字面量 **`布局线条图`** | 补标题 |
| U4 | 线条图代码块内含框线字符（至少一个 `┌` 或 `+--` 或 `│`）且行数 ≥5 | 重画，禁止空块/三行糊弄 |
| U5 | 线框中的主控件能在「关键组件与交互」表找到对应行 | 对齐补全 |

**确认 REQ/UID 前**：有 UI 的每个 UID 须 U1–U5 全过；不过 → 禁止标「已确认」。

## 反模式

| ❌ | ✅ |
|----|-----|
| §2 只有区域表、无线框（旧「可选」写法） | **区域表 + ASCII 布局线条图** 缺一不可 |
| 线框只有「顶栏/内容/底栏」三个字 | 画出输入框、按钮、列表行等占位 |
| REQ 里写 `#FF5733`、具体 px 间距 | 写「主按钮、表单区」，样式待定 |
| REQ 直接写 `contracts/UI-001` | 写 `requirements/ui/UID-001` |
| 有页面却不建 UID | 评估后建 UID 或 REQ 注明「无 UI」 |
| Agent 替用户定配色/字体 | 标待定 + humanTodo 或 sol AskQuestion |
| FE dev 3.1 只有组件名表、无线条图 | 3.1 含线框（对齐 UID）+ 区域/组件表 |
| FE dev 映射表写中文「体重」 | 写 API 英文字段 ``value`` / ``recordedAt`` |
