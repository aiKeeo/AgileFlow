# 任务跟踪

## 作用

自动生成并全程维护代办事项列表，让整个开发过程可跟踪、可追溯。

## 执行流程

1. 基于 `specs/003-solution.md` 中的原子任务列表，生成初始 `specs/todo.md`
2. 每个环节完成后，自动更新对应流程任务的状态
3. 开发过程中，每完成一个开发子任务立即更新状态
4. 所有任务完成后，生成 `specs/completion-report.md`

## 强制输出格式（specs/todo.md）

```markdown
# 项目代办事项

## 项目信息
- 项目名称：
- 创建时间：
- 预计完成时间：

## 流程进度
- [ ] 需求澄清
- [ ] 数据建模
- [ ] 技术方案
- [ ] 开发实现
- [ ] 测试验收

## 开发任务
- [ ] 任务1：创建数据库表（30min）

## 进行中
- 无

## 已完成任务
- 无

## 变更历史
| 时间 | 操作 | 说明 |
|------|------|------|
```

## 状态标记规范

| 标记 | 含义 |
|------|------|
| `- [ ]` | 未开始 |
| `- [ ] 🔄` 或单独「进行中」区列出 | 进行中 |
| `- [x]` 或 `✅` | 已完成 |

**规则**：任务完成后从「开发任务」移到「已完成任务」，并追加「变更历史」一行。

## 强制文档产出

| 时机 | 文件 |
|------|------|
| 初始化 | `specs/todo.md` |
| 每次状态变化 | 更新 `specs/todo.md` |
| 全部完成 | `specs/completion-report.md` |

## 强制规则

- 任何任务完成后，必须立即更新 `specs/todo.md`
- 禁止跳过任务直接标记完成
- 所有更新必须在「变更历史」中保留记录
