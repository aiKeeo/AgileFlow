# Agileflow 交互示例

> 演示**节奏**（谁停、落什么）；业务名可换。  
> **默认路径**：契约空着 → **启动卡问人**。**明确委托**（你定/别问我）才走 `AF_DECIDE=ai`。`user` 分步停见示例 **C**。  
> 排障 → [TROUBLESHOOTING](../TROUBLESHOOTING.md)

---

## A. 新手第一次用（默认：问人）

**用户**：我想做个小程序待办，不太懂流程，怎么开始？

**AI**：AskQuestion **流程启动卡**（我来决策 / AI 全权）→ **停**（不写 REQ；env 仍 pending）  
**用户**：选「AI 全权决策」  
**AI**：写 `agileflow.env`（`AF_DECIDE=ai`）→ 落盘 REQ → 过闸 → 摘要（免卡）→ **同回复连做**建模判定/sol → …（写码仍须①）

---

## B. 明确委托：AI 全权（可连做）

**用户**：做账号密码登录，你定，别问我，直接做完。

**AI**：
```
📍 Agileflow | 决策：AI全权 | 契约：明确委托
```
写 env → 派 role-req → REQ（含 AC）→ `req-confirm` 绿 → **总控标 REQ 已确认** → 一行摘要 → **同条**派 role-model（跳过判定）/ role-sol …

> 少问 + 可连做 ≠ 跳主链；每阶段仍须闸门；①与可运行闸门仍强制。
> **若用户只说「做登录」未说「你定」** → 走示例 A（启动卡），不是本路径。
---

## C. user 我来决策（对照：更多停点）

**用户**：`req:` + 描述（契约已是 `AF_DECIDE=user`）  
**AI**：缺口卡（若有）→ **停** → 写 REQ → 确认卡 → **停** → 阶段闸门 → **停** → 用户「是，继续」→ 下条进下一阶段。

---

## C2. 中途 AI 接管（曾选 user → 随时可切）

**契约**：曾是 `user`，用户已走完 1～2 阶段，不想再盯了。  
**用户**：不想看了，后面都你定，剩下你来。  
**AI**：改 env → `AF_DECIDE=ai` → **同条或下条连做**剩余阶段（仍须每阶段过闸门）。

> 一开始选「我来」**不锁死**；随时可让 AI 接管，也可再说「这阶段我来」收回。

---

## D. 探索（不知道做什么）

**用户**：项目好慢，想优化，不知道瓶颈在哪。  
（或 `explore: 看看性能`）

**AI**：
```
📍 Agileflow | 探索模式
```
读代码 → 发现摘要 + 2～4 个方向 → AskQuestion → **停**（**不写 REQ/源码**）  
**用户**：选「先治接口 N+1」  
**AI**：再进 `req:` / 豁免，按正式流程。

---

## E. 中途改需求（L2）

**用户**：登录成功不要跳首页，改跳上次浏览页，AC 要改。

**AI**：`纠偏：L2 → REQ-001` → 改 REQ AC（按 change-management 留痕）→ 影响分析卡 → **停**  
**用户**：确认只动 FE  
**AI**：相关 T 重做③（必要时回①）→ 再验收。

---

## F. 并行 FE+BE

**用户**：登录前后端并行，用 subagent。

**AI（`ai`）**：进阶段 4 扫描 → 发现 T-001-BE + T-002-FE（同 F、无 depends_on、路径分离）→ 首行 `并发：自动并行 T-001,T-002（FE+BE）` → 同时派 2 个 role-dev → 回报后逐 gate 勾。

**AI（`user`）**：扫描到同上候选 → **并行启动卡** → **停** → 确认后同上。

> `user` 未开并行卡 → 禁止 Subagent 写业务码。

---

## G. 豁免（修一行 · 仅非 AF 项目）

**用户**：首页标题拼错一字，不走流程（项目**无** `atlas/agileflow.env` / `requirements/`）。

**AI**：`📍 豁免：微型` → 直接改源码 → 不做 REQ/dev。  
已启用 AF / 多模块/API/DB **禁止**冒充豁免。

---

## H. humanTodo 阻塞 → 解除

**AI**：（sol）写入 humanTodo「提供支付商户号」→ 可标阻塞  
**用户**：已写入 `.env`  
**AI**：勾掉 humanTodo → 继续被阻步骤。

---

## I. 阶段 4 单 T：一次派活 · 内 ①→②→③

**用户**：做 T-001 登录 API。

**总控**：
1. TodoWrite：①构思 / ②写码 / ③AC  
2. **派 1 次** role-dev（任务：T-001 · 内须 ①→②→③ 完整交付）  
3. 收回报 → 总控：`dev-step1-literal` 绿→勾① → `write-code` 绿→勾② → 证据/`--only todo` 绿→勾③  
4. 全部 T 齐 → `dev-complete`

禁止：总控自己写 dev；跳步先码后补；未回填 `## 结果` 勾③。

---

## J. 验收报告

**用户**：`tests:`  

**AI**：入场门禁（同会话可复用③）→ 写 `atlas/tests/REQ-*-验收报告.md` → 更新 `tests/README.md` → PASS / BLOCKED-HUMAN。

---

## K. 已有：方案 / init / 续跑（短）

| 场景 | 节奏 |
|------|------|
| **sol**（user） | 技术栈卡→停 → `architecture.md` + `atlas/todo.md` → 确认/闸门→停 |
| **sol**（ai） | 自选栈落盘 → 摘要/免卡 → **可连做**进 dev① |
| **init:** | 写法锚点卡→停 → 扫描落盘 init → 确认→停 |
| **继续 agileflow** | 读 `todo.md` → `## 进行中` **checkpoint**（`T-xxx · 步骤 ①|②|③`）→ TodoWrite 对齐 → Read 磁盘代码 → 从下一步续（`ai` 可连做） |

---

## 违规反例

- ❌ 探索类问题直接写 REQ/源码  
- ❌ 首启未问就写 env/REQ  
- ❌ 无①写码；事后补写勾①  
- ❌ 勾满①②③却无 `dev/` 或无可运行证据  
- ❌ 「用户催了」跳过闸门  
- ❌ `ai` 却每阶段停等「继续」（应连做）  
- ❌ `ai` 派完一批 Subagent 就交班、等人说「继续」（应阻塞等回报后同会话下一批）  
- ❌ 连做时跳过闸门或薄写 sol·dev  
- ❌ 标「开发实现 ✅」/进 phase5，但 REQ AC 仍「（③ 后填）」
