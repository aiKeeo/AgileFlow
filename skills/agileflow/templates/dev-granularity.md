# dev 颗粒度 SSOT（全端 · 叙述五段式）

> FE/MP 范例：[dev-exemplar-FE.md](../examples/dev-exemplar-FE.md) · BE 范例：[dev-exemplar-BE.md](../examples/dev-exemplar-BE.md)  
> 更多例子：[dev-reuse-examples.md](../examples/dev-reuse-examples.md)

## 文档结构（全端统一）

| 段 | 写什么 |
|----|--------|
| **摘要** | 本 T / 做(一行索引) / 不做 / 上游 / AC |
| **主流程** | 从入口事件编号叙述 happy path（3～6 步） |
| **边界** | 特殊情况：在第几步拦截、怎么处理 |
| **实现说明** | 每个【新写】【改动】：目的 · 做什么 · 怎么做 |
| **结果** | 可运行证据 + AC 映射 |

**FE/MP** 主流程主语：用户动作 → method。  
**BE** 主流程主语：请求/回调/定时 → Controller → Service → 返回/落库。

---

## 1. T 拆分

| 原则 | 说明 |
|------|------|
| 一切口一 T | 1 T = 用户可独立完成的一个入口流程，或 BE 的一个 API 切片 |
| 对齐 F/UI/API | 1 T 对应 1 个 UI-xxx 页面主流程或 1 个 API-xxx |
| 禁止合文件 | BE+FE 不得同文件；多 T 不得糅合 |

**口诀**：用户要不要换页面/换入口？换 → 新 T。

---

## 2. 主流程

| 项 | 标准 |
|----|------|
| 入口 | 首行 `> 入口：…`（FE：用户进入…；BE：POST /api/…） |
| 步数 | 标准档 **3～6** 步；完整档 ≤8 |
| 一步 | FE：一个用户动作；BE：一次请求链上的编排节点 |
| 写法 | FE：`用户… → method()`；BE：`请求进入 → Controller.method()` |
| 禁止 | 从 repository 写起；mermaid；dev 内字段映射表；`## 步骤` 旧格式 |

---

## 3. 边界

| 项 | 标准 |
|----|------|
| 位置 | 独立 `## 边界` |
| 一条一边界 | 空态 / 冲突 / 401 / 幂等 / 库存不足 / 连点 各一条 |
| 挂钩 | 写「第 N 步」或「`handler` 内」 |

---

## 4. 实现说明

| 项 | 标准 |
|----|------|
| 展开谁 | 主流程+边界出现的 **【新写】【改动】** |
| 复用 | 【复用】可简短：目的 + 本 T 怎么用 + 不改什么 |
| 必含 | **目的** + **做什么** 或 **怎么做**（含 if/else 编号） |
| 算法 | 链 API-xxx，不粘贴大段 JSON |
| 字段 | FE 链 UI-xxx §字段绑定；BE 链 API-xxx |

块标题格式：`### \`path/to/file\` 【新写|改动|复用】` 或 `### \`ClassName.java\` 【新写】`

---

## 5. 摘要「做」与实现说明

```
主流程（组件/类 + 方法）
    ↓ 标注新写/改动/复用
摘要「做」（一行索引）
    ↓ 逐项展开
实现说明（目的+逻辑）
```

---

## 6. 档位

| 档位 | 主流程 | 实现说明 |
|------|--------|----------|
| minimal | ≥3 步 | 仅【新写】块 |
| standard | 3～6 步 + 边界 ≥2 条 | 新写+改动全展开 |
| full | 同 standard；算法可更深 | 同 standard |

`AF_DECIDE=ai` / `fast` **不减**主流程与实现说明厚度。

---

## 7. 写码约束

- ② 写码：Write ⊆ `## 实现说明` 中【新写】【改动】落点（**全端统一**）
- 勾 ① 前：`## 主流程` + `## 边界` + `## 实现说明` 写满；仅 `## 结果` 可写码后填
