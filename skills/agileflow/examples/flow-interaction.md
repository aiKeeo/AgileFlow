# Agileflow 交互示例

## 意图识别示例

**启用**：用户 @agileflow 或说「走完整流程」「继续 agileflow」
**不启用**：「这段代码什么意思？」「帮我 review 一下」→ 直接回答，不走五阶段

**用户**：「帮我把订单支付 API 写了」（无 specs、未提流程）
→ **不默认启用**；可先答疑或 AskQuestion 是否走 Agileflow

**用户**：「@agileflow 帮我把订单支付 API 实现了」
→ 读 003/todo → 有方案、开发任务未完成 → **阶段 4**
```
📍 Agileflow | 模式：严谨 | 阶段：4-开发实现 | 依据：用户要求实现支付 API，003 已存在
```

**用户**：「继续 agileflow」
→ todo：需求✅ 建模✅ 方案⬜ → **阶段 3**
```
📍 Agileflow | 模式：快速 | 阶段：3-技术方案 | 依据：todo 下一未完成阶段
```

**用户**：「我想做个旅游 App」（无 specs/）
→ 全新项目 → **阶段 1** → AskQuestion 需求卡片

**用户**：「写需求」但 REQ 已全部已确认
→ 意图与状态冲突 → **AskQuestion 阶段确认**（改 REQ / 进阶段 2 / 其他）

## 正确示例（完整流程）

**用户**：我想开发一个旅游规划软件，不想每次旅行前都规划。

**AI（Agileflow）**：
```
📍 Agileflow | 模式：快速 | 阶段：1-需求澄清 | 依据：新功能，无 specs/
已启动 Agileflow，判断为【快速模式】（个人工具，可先做 MVP）。
```
*（读取 phases/01-requirement.md → AskQuestion 需求卡片 → 停止）*

**用户确认需求后**：
```
✅ 需求澄清已完成。
生成需求文档：REQ-001 一键复用行程, REQ-002 偏好记忆（共 2 个）。
```
*（AskQuestion 阶段闸门："是否继续进入【数据建模】？" → 停止）*

**阶段 4 开发完成后**：
```
✅ 开发实现已完成。
```
*（AskQuestion："是否进入【测试验收】？" → 停止）*

**阶段 5 完成后**：
```
✅ 测试验收已完成。
交付汇总：specs/tests/README.md → PASS
```
*（流程结束）*

## 反例（禁止）

- ❌ 自然语言"请问是否继续？"而不 AskQuestion
- ❌ 阶段 4 直接跑验收测试
- ❌ AskQuestion 后同一回复继续下一阶段
