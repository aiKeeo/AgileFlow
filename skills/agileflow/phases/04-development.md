# 阶段 4：开发实现

> **执行清单（先读）**：[dev-quickstart.md](../templates/dev-quickstart.md)  
> 模板：[dev-rationale.md](../templates/dev-rationale.md) · 金标准：exemplar-BE / exemplar-FE · 写法锚点：[code-conventions.md](../templates/code-conventions.md)

## 规则（构思先行）

| 规则 | 说明 |
|------|------|
| **无构思禁写码** | 无合规 dev → 禁止 Write 业务源码 |
| **构思须落盘** | 构思必须落盘；聊天不算 |
| **一任务一文件** | 1 T = 1 文件；禁止多 T 糅合 / 合并勾选 |
| **九段标题全齐** | **快速/严谨均须九段标题全齐**；快速二/四/六/七可 `→ 引用`+待补齐；严谨须引用+增量；禁止摘要/空壳/省略标题 |
| **五须可执行** | 五 = 目的→需要什么→5.x（做法含类·方法或页面路径） |
| **②③分工** | ② 只执行五；③ 只验八；催进度不豁免 |
| **FE 须有 3.x** | FE+UI：须有字面量 `### 3.1` `### 3.2` `### 3.3`；**3.1 含 ASCII 布局线条图**（对齐 UID，禁纯表）；3.2 英文字段 |
| **默认串行** | **默认串行**：一次只推进 1 个 T；禁止批量写完所有①再写码。**例外**：用户显式并行 **且** 已过 [parallel 开闸卡](parallel-orchestration.md#并行开闸卡强制) **且** 走构思齐批；单次构思齐批 **最多 3 个 T** 的①；仍禁止无并行许可的批量① |
| **连续做≠压缩** | 连续做 ≠ 压缩模板 |
| **补盘不勾①** | 补盘/摘要/空壳 → 不勾 ①；父任务不得洗成正式 ✅；**即使用户说「接受技术债」也禁止**把「开发实现」标 ✅（只允许标 `⚠️ 补盘归档`，须另开正式①重做才能 ✅） |
| **构思闸门全过** | 构思闸门：九段标题 + 五可执行 + **反假九段** + **字面量校验**；有文件但 grep 不过 → **禁止勾①** |
| **强制严谨条件** | `todo` 中 **T≥3** 或同时有 **BE+FE** → **强制严谨**（见 flow-modes） |
| **禁止伪并行** | 「全部开发」≠ 并行；**禁止**无显式并行就启 Task/Subagent 写业务源码；1 Task=1 T 且须已有合规① |
| **开发完成格式门槛** | 标「开发实现 ✅」前须**同时**满足：① 合法 T 头（`^#{3,4} T-\d+`，**仅 ###/####**）≥1；② 每个 T 头下有 ①②③ 三行（含 `atlas/dev/T-xxx` 路径）；③ `count(atlas/dev/T-*.md)`（非⚠️、非 README、非 temp/）= 合法 T 头数；④ 每 T 可运行闸门证据齐。扁平列表或有头无三段式 → **格式违规禁止 ✅** |
| **TodoWrite 强制展开** | 进阶段 4 /「全部开发」：**先 TodoWrite 展开**——每个 T **必须三条**「①构思 / ②写码 / ③AC」；禁止一条 Todo 多 T；未展开禁止 Write 业务源码（见 [todo.md](../templates/todo.md#todowrite-强制展开防漏①--最高优先级)） |
| **可运行闸门** | 每 T 勾③前必须 **编译 + 能启/能调接口 + 本 T 冒烟**；F-xxx/MVP 切片完再跑模块冒烟；「给用户看」前存在端启动冒烟全过；证据写入 dev 九；单测绿 ≠ 可交（见 [dev-quickstart 可运行闸门](../templates/dev-quickstart.md#可运行闸门每-t--每模块强制)） |
| **像素对比**（有强制原型） | 勾③前须 [fe-pixel-compare](../templates/fe-pixel-compare.md) PASS；闸门扫 report |

催进度反模式 → [SKILL.md](../SKILL.md)。Subagent 细则 → [parallel-orchestration.md](parallel-orchestration.md)。

## 严谨 vs「全部做 / 直接全开发」

| 模式 | 「全部做 / 连续做 / 直接全开发」时 |
|------|-----------------------------------|
| **快速** | 仅未命中强制严谨时；任务间连续 **完整** ①→②→**可运行闸门**→③；仍过构思闸门/写码闸门/可运行闸门 |
| **严谨** | 不适用连进压缩；逐 T **完整一～九**；可连续，不可摘要/空壳；仍过可运行闸门 |

**客观强制严谨**（命中任一 → 必须严谨，用户说「快速」也拒绝）：

- `atlas/todo.md` 开发任务 **≥3 个** T 头（行匹配 `^#{3,4} T-\d+`，仅 `###`/`####`）
- 或同时存在 `[BE]` 与 `[FE]` 任务
- 或支付/权限/DB·服务端/用户数据/多模块/>1 天

## 三段闸门（构思 / 写码 / 可运行）

```
Read exemplar → 写 1 个 T 的 dev →【构思闸门】→ 勾 ① →【写码闸门】→ ② →【可运行闸门】→ ③
（禁止：先写完 T-002～T-011 的 dev 再统一写码；禁止跳过可运行闸门）
```

### 构思闸门 — 勾 ① 前

| 检查项 | 过线标准 | 不过 → |
|--------|----------|--------|
| **读金标准** | 已 Read 本端 exemplar（BE→exemplar-BE / FE→exemplar-FE） | 先 Read |
| **串行约束** | **默认串行**：仅 1 个进行中 T；无其他 T「①已勾且②未完成」。**例外（显式并行）**：开闸卡已确认且走 [parallel](parallel-orchestration.md)——**构思齐批** 允许多个 T「①已勾、②未开」（**≤3**）；**写码执行批** 允许多个 T **同时**处于②/③（1 Task=1 T）。**仍禁止**：无开闸时出现多份「①勾完②未完」；「全部开发」冒充并行 | 串行：先完成或回滚；伪并行：改串行 |
| **文件存在** | `atlas/dev/T-xxx-*.md` 存在 | 先写文件 |
| **九段标题** | **段标题**过线（见下） | 补标题 |
| **五可执行** | **五可执行**过线（见下）；FE+UI 另过 **3.x 字面量** | 重写五 / 补 3.1–3.3 |
| **非补盘** | 非补盘/非摘要；无 `顺序：⚠️` | 标 ⚠️；重写 |
| **反假九段** | **反假九段**过线（见下） | 按 exemplar 重写整份 |
| **字面量校验** | **机械 grep**全过（见下）；须有命中证据 | 补字面量后重 grep；**禁止勾①** |

#### 九段标题（可 grep）

| 模式 | 须存在 |
|------|--------|
| **严谨** | `## 一、` … `## 九、` **全部**（字面量含 `## `） |
| **快速** | 九段标题全齐；二/四/六/七 可 `→ 引用`+**待补齐**；一/五/八/九 + FE 三 须实质 |

五内须有：`### 目的` `### 需要什么` `### 怎么做` ≥1× `#### 5.`

**禁止**：用纯文本 `一、` `二、`…（无 `##`）冒充段标题——这是 **假九段**，九段标题/反假九段检查不过。

#### 五可执行（防空壳标题）

每个 `#### 5.` 小节须**同时**满足：

1. 含字面量 **要达成什么** / **需要什么** / **做法**（或同等加粗标题）
2. **做法**下 ≥2 条编号步骤
3. 做法中 ≥1 处反引号代码痕迹：`` `ClassName.method` `` 或 `` `pages/...` `` 或 `` `services/...` ``
4. **禁止**五的正文（目的/需要什么/做法）整段仅为「见 REQ」「见 API」「见 contracts」「实现即可」
5. **禁止**用「五、关键代码落点：Xxx.java」一句话代替整个五（无 `### 目的` / `#### 5.` = 不过）

FE+UI 另须可 grep：`### 3.1` `### 3.2` `### 3.3`；3.2 表含英文契约字段列；**3.1 正文须含布局线条图**（代码块内有 `┌`/`│`/`+--` 之一，且非空）。

#### 反假九段（bbb 事故形态 · 机械）

勾①前 Agent **必须自检**（任一项命中 → **禁止勾①**，重写）：

| 检查项 | 命中即失败 |
|--------|------------|
| **纯文本假标题** | 全文几乎无 `## 一、` / `## 五、`（只有 `一、`/`五、` 纯文本） |
| **缺五结构** | 无 `### 目的` 或无 `#### 5.` |
| **五无代码落点** | 「五」无 `` `Class.` `` / `` `pages/` `` / `` `services/` `` 痕迹 |
| **FE 缺布局** | FE+UI 无 `### 3.1` 或 3.1 无线条图字符 |
| **补盘冒充正式** | 文中出现「② 写码完成 / 已写码 / 等待联调」却仍要勾**正式①**（属补盘或事后摘要） |
| **文档过短** | 全文过短：去掉空白后少于约 800 字符，且无 ≥2 个 `#### 5.`（纯脚手架可 1 个 5.x，但仍须有完整五结构，不得假九段） |

**金标准**：与 exemplar 同结构；颗粒度可略简，**结构不可降级成摘要**。

#### 字面量校验（勾①前强制 · 假构思勾不了）

> **不靠记住。** 勾①前必须对本 `atlas/dev/T-xxx-*.md` **实际 Grep**；缺字面量或命中禁形 → **禁止** TodoWrite①=`completed` / todo①=`[x]`。  
> 速查表 → [dev-quickstart 字面量校验](../templates/dev-quickstart.md#字面量校验勾①前强制--假构思勾不了)。

**必过字面量**：`## 一、` · `## 五、` · `### 目的` · `### 需要什么` · `### 怎么做` · `#### 5.` ·（`` `Class.` `` 或 `pages/` 或 `services/`）  
**FE+UI 另加**：`### 3.1`（含 `┌`/`│`/`+--`）· `### 3.2` · `### 3.3`  

**禁形（ai-error-case）**：`## 一、目标` · `## 五、可执行方案` · `## ① 构思` / `## ② 关键实现点` —— 出现且缺「目的/5.」结构 → 不过。

勾①时回复须含一句：`字面量校验：通用项已命中[ +FE布局/字段表]`（可附路径）。无此句视为未过构思闸门。

### 写码闸门 — Write 源码前

| 检查项 | 过线标准 |
|--------|----------|
| **① 已勾** | 该 T 的 ① 已勾 |
| **单任务约束** | 仍只做一个 T；Write ⊆ 五当前 5.x |
| **读过写法锚点** | 已 Read **本端**写法锚点（`p1-frontend|backend` 或 `code-patterns-*`）——**至少含资产索引**；严谨另对齐 §三 |
| **复用盘点就绪** | 有 UI/API 时：复用盘点已填；表写「复用」则 5.x/代码须用该路径，禁止平行实现 |
| **严谨资产对齐** | **严谨**：七含具体 `资产索引`行 + `§3.x`；**快速**：读过锚点 + 复用盘点即可 |

### 可运行闸门 — 勾 ③ / 父任务 ✅ 前

| 检查项 | 过线标准 | 不过 → |
|--------|----------|--------|
| **编译构建** | 本 T 涉及端 **编译/构建**成功（静态检查+构建） | 修编译，禁止勾③ |
| **能启能调** | 按 [能启能调分档](../templates/dev-quickstart.md#能启能调分档堵单测绿但起不来--不改流程)：**该端第 1 个 T 须真启动**；后续单 T 可 MockMvc；**F/MVP 切片与给用户看须真实启动** | 修启动与配置 |
| **主路径冒烟** | 本 T 主路径冒烟不炸（不 500；核心读写走通） | 修功能 |
| **证据已写** | 证据已写入 dev **九** 或 todo（命令+结果） | 补证据 |

权威细则与「给用户看 / 模块级」时机 → [dev-quickstart 可运行闸门](../templates/dev-quickstart.md#可运行闸门每-t--每模块强制)。  
阶段 5 **测试入场门禁** **不能替代**本闸门。

## ①②③ 要点

- **①** 模板 → [dev-rationale](../templates/dev-rationale.md)；照 exemplar 颗粒度  
- **②** 只把五变成代码；FE 键名对齐 3.2  
- **③** 先过 **可运行闸门**，再只测八；九回填（含可运行证据 + AC单测结果）；父任务 ✅  

### 默认串行（禁止批量① / 禁止 Subagent 外包）

0. **TodoWrite 展开**（TodoWrite强制展开）：为每个 T 建 **①②③ 三条**（强制）；「全部开发」时①条目须先全部出现为 pending  
1. 取**一个** ①未勾 的 T → 进行中  
2. 读金标准→写该 T 的 dev→构思闸门→勾①（TodoWrite + atlas/todo）→写码闸门→②→**可运行闸门**→③  
3. **在该 T 的 ② 开始前，禁止**再为其他 T 写 dev 或勾①（**显式并行构思齐批 除外**，见下）  
4. 「全部开发」时重复 1–3，**仍由主 Agent 串行**；禁止用 1～2 个 Subagent「实现全部 BE/FE」；**禁止跳过可运行闸门**  
5. 全部父任务 ✅ **且** `dev` 文件数 = T 头数 **且** 每个 T 的 TodoWrite① completed **且** 可运行闸门 证据齐 → 结束闸门 → 停  
6. 任一 **F-xxx / MVP 切片** 相关 T 全 ✅ → 模块编译+冒烟（可运行闸门）→ **AskQuestion 阶段性确认卡** → **停止**
7. 用户要「给用户看」或选「先给我看」→ 存在端全量启动冒烟过线 + 给出启动步骤；等用户反馈  
8. 全部 T ✅ → 阶段闸门（进 tests）→ 停；prompt 须含可运行闸门 证据摘要 

### 显式并行时（与「串行约束」 的关系）

仅用户原话含「并行 / subagent / 同时开 FE+BE」时启用 → [parallel-orchestration.md](parallel-orchestration.md)：

| 批次 | 允许 | 仍禁止 |
|------|------|--------|
| B（齐①） | 多个 T 先各自写合规 dev、过 A、勾① | 「全部开发」冒充并行；无①就② |
| C（并行②） | 1 Task = 1 T，且该 T ①已勾 | 一个 Subagent 多 T；Subagent 写①或改 todo ✅ |

**① 只由主 Agent 写**；Subagent **只做②→可运行闸门→③**。
