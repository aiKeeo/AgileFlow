## 序（不可跳、不可批量①、不可 Subagent 外包、不可交不能跑的码）

```
【默认·主 Agent 串行】
0. 确认 atlas/todo 已是 ### T-xxx + ①②③（扁平 `- [ ] T-001` 须先改写，禁止直接写码）
1. TodoWrite 展开：每个 T **三条**「①构思 / ②写码 / ③AC」
   ※ 未展开 → 禁止 Write 业务源码（见 todo.md「TodoWrite 强制展开」）
2. 取下一条未完成的① → Read exemplar → 写【该】T 完整 `atlas/dev/T-xxx-*.md`
   →【闸门 A】→ 勾 ①（TodoWrite + atlas/todo.md）→【闸门 B】→ ②
   →【闸门 C 可运行】→ ③
3. 下一条①再重复。禁止先写完多份 dev 再统一写码（无并行许可时）。
   ※ `dev/README.md` 索引 ≠ ①；每个 T 必须独立文件
```

**连续做 / 全部开发 / yes_all** = **先展开 TodoWrite**，再按清单逐条做（每条含闸门 C）；**≠** 摘要 / 空壳 / 合并多 T / **从不编译启动** / **跳过 TodoWrite** / **扁平 todo**。

| 用户说 | 主 Agent 必须 | 禁止 |
|--------|---------------|------|
| 全部开发 / 直接全开发 | **先**确认三段式 todo → TodoWrite 为每个 T 建① → 串行 ①→②→C→③ | 启 Task 批量写码；扁平 `- [ ] T-001`；1 个 Todo 多 T；无①就写码；跳过闸门 C；只用 README 假构思 |
| 并行 / 多 subagent（显式） | 先读 [parallel-orchestration](../phases/parallel-orchestration.md)；每 T 先有合规①；②后仍过闸门 C | 无 dev 就让 Subagent Write；Subagent 跳过 C |
| 给用户看 / MVP 演示 | 存在端编译+启动+MVP 主路径冒烟全过 | 「代码齐了」就甩给用户 |

**未读到用户原话含「并行/subagent」→ 禁止启用 A1 并行例外。**

**标「开发实现 ✅」前**：R16 全过——合法 T 头（`^#{3,4} T-\d+`）≥1 **且** 每头下有①②③ **且** `dev/T-*.md` 数 = 头数（**README/temp 不算**）**且** TodoWrite①②③ completed **且** 每 T 闸门 C 有终端证据。  
若开发任务区有 `T-\d+` 但合法 T 头=0（扁平列表）或有头无三段式 → **禁止 ✅**。

### A7 机械 grep（勾①前强制 · 假构思勾不了）

> **不靠记住，靠卡死。** 勾 TodoWrite①=`completed` 或 `atlas/todo` ①=`[x]` **之前**，必须对本文件跑 Grep（或等价），**全部必过项命中**；回复中须贴出命中证据（路径+关键行）。  
> 权威细则亦可对照 [04 A7](../phases/04-development.md#a7-机械-grep勾①前强制)。

**必过（BE/FE 通用）**——缺任一 → **禁止勾①**：

| # | 文件中须存在字面量 |
|---|-------------------|
| G1 | `## 一、` |
| G2 | `## 五、` |
| G3 | `### 目的` |
| G4 | `### 需要什么` |
| G5 | `### 怎么做` |
| G6 | `#### 5.` |
| G7 | `` ` `` 内含 `.` 的类方法 **或** `pages/` **或** `services/`（至少一处） |

**FE+UI 另必过**：

| # | 须存在 |
|---|--------|
| G8 | `### 3.1` 且正文含 `┌` / `│` / `+--` 之一 |
| G9 | `### 3.2` 与 `### 3.3` |

**命中即失败（ai-error-case 薄稿形态）**——出现且缺 G3–G6 → **禁止勾①**：

| 禁形 | 说明 |
|------|------|
| `## 一、目标` | 不是模板「一、需求理解」 |
| `## 五、可执行方案` | 不是「五、核心流程」+ `### 目的`/`#### 5.` |
| `## ① 构思` / `## ② 关键实现点` | 用①②③当章节冒充九段 |
| `## 八、验证` 且无 `## 八、REQ` 类验收表头、且全文无 `### 目的` | 薄验证段冒充八 |

pattern: ## 一、|## 五、|### 目的|### 需要什么|### 怎么做|#### 5\.|### 3\.1|### 3\.2|### 3\.3

### 段标题

- **严谨**（T≥3 或 BE+FE 或 DB/权限等 → 强制）：`## 一、`～`## 九、` 全有  
- **快速**：`## 一、` `## 五、` `## 八、` `## 九、`（FE+UI +`## 三、`）；二/四/六/七**可省略整段**

五内：`### 目的` `### 需要什么` `### 怎么做` `#### 5.`

### 五可执行（防空壳）

每个 `#### 5.`：

- 有 **要达成什么 / 需要什么 / 做法**
- 做法 ≥2 步，且含 `` `Class.method` `` 或 `` `pages/...` `` / `` `services/...` ``
- 禁止五的正文只有「见 REQ / 见 API / 实现即可」
- 禁止「五、关键代码落点：Xxx.java」一句话顶替五

FE+UI：须有 `### 3.1` `### 3.2` `### 3.3`；**3.1 须含 ASCII 布局线条图**（对齐 UID）

## 闸门 C（可运行 · 每 T / 每模块强制）

> **写完码 ≠ 能用。** 勾 ③、父任务 ✅、或说「可以给用户看」之前，Agent **必须亲自跑终端**过线。  
> 命令以 `architecture.md` 为准；无约定用下表默认。

### 何时跑

| 时机 | 范围 |
|------|------|
| **每个 T** 在 ② 写完、勾 ③ 前 | 本 T 涉及的端 + 本 T 主路径 |
| **一个 F-xxx / MVP 切片** 相关 T 全 ✅ 时 | ① 该模块编译 + 模块冒烟（闸门 C）→ ② **AskQuestion**（含是否 Playwright FE 冒烟，见 [fe-smoke-playwright](fe-smoke-playwright.md)）→ 阶段性确认卡 → 停 |
| **给用户看 / MVP 演示 / 交付体验** | 存在端全量：编译 + 启动探针 + MVP 主路径冒烟 → **可询问** FE Playwright → 再问用户或给出启动步骤 |

### 过线标准（须同时满足适用项）

| # | 检查 | BE | FE/小程序 |
|---|------|----|-----------|
| **C1 编译** | lint/type 无 error + build 成功 | `mvn -q -DskipTests package`（或约定） | `build` / `build:weapp` 成功 |
| **C2 能启/能调** | 见下方「C2 分档」（**不削弱**原可运行要求） | 见下 | 见下 |
| **C3 冒烟** | 本 T（或本模块）主路径不炸 | 该 T 的核心写/读接口：200/业务成功，不 500 | 该页能打开、核心按钮能走完（可人工点，须记录结果） |

#### C2 分档（堵「单测绿但起不来」· 不改流程）

| 场景 | C2 最低标准 |
|------|-------------|
| **该端（BE 或 FE）本阶段第 1 个 T 勾③前** | **必须真实启动**：BE 进程+health/探针 UP；FE `dev` 能起或开发者工具可开 |
| **同端后续单 T** | BE：health UP **或** 本 T API 用 curl/MockMvc/集成测打通 happy path（非 500）；FE：能启或关键页不白屏 |
| **F-xxx / MVP 切片确认前** | **必须真实启动**（同第 1 个 T） |
| **给用户看 / MVP 演示** | **必须真实启动**；**禁止**仅用 MockMvc 代替 |

> 同端第 1 个 T 强制真启，避免「一路 MockMvc 绿、切片才发现起不来」。中间 T 可用 MockMvc 保节奏；切片/演示仍须真启。


**与 ③ / 5-0 的关系**：

| | 闸门 C | 步骤 ③（AC） | 阶段 5 的 5-0 |
|--|--------|--------------|----------------|
| 目的 | **能编、能启、主路径不炸** | Given/When/Then **对不对** | 进 tests 前**全量**再验一遍 |
| 深度 | 编译+探针/接口+轻量冒烟 | 1 AC ↔ 1 自动化测试 | G1+G2+G3 全端 |
| 失败 | **禁止勾③ / 禁止说可给用户看** | 回修代码 | 禁止 5A |

### 证据（勾③前必须有）

写入 dev **九** 或 `atlas/todo.md` 该 T 下：

- C1/C2/C3 各一行：命令 + 结果（exit 0 / health UP / 冒烟通过）
- 若跑了 FE Playwright：附 `atlas/logs/fe-smoke-report.json` 的 summary（pass/fail/skip）
- **禁止**只写「测过了」无命令无结果

### ❌ 不过

- 只跑 `test/ac` 单测、从不 `package` / `build`  
- 编译红仍标 T ✅  
- 服务起不来 / 登录 500 / 首页白屏仍说「MVP 好了给用户看」  
- 把闸门 C 推到「等阶段 5 再说」
